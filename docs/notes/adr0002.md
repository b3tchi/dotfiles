# ADR [[cat001]] [[product]]

## title
When to use bash (vs nushell)

## context
[[adr0001]] established nushell as the default scripting language and bash as the fallback "when spawning nushell is not worth it". That phrasing leans on `POSIX-required` as one of the carve-outs, which is too elastic — almost any shell script can be argued POSIX-required if you squint. Without a concrete test, the bash fallback silently grows and erodes [[adr0001]] in practice.

This ADR pins down the exact conditions under which bash wins. If a script does not meet at least one, the default ([[adr0001]]) applies and it is nushell.

## decision
Use bash when **at least one** of the following is true. Otherwise, default to nushell per [[adr0001]].

1. **Runs before nushell is installed or loaded** — bootstrap scripts (`install-rotz.sh`, `scripts/ubuntu/*.sh`), package-manager hooks, anything executed by a fresh OS image before `rotz install` completes.
2. **System-shell startup chain** — `profile`, `distro/bashrc`, anything sourced by bash itself before tmux starts. Nushell does not exist yet at this layer.
3. **Tmux outer-shell wrapper** — the bash → tmux → nushell handoff (`start_local_session`, workspace detection in `bashrc`). Bash is the parent of tmux; nushell is the child. The wrapper stays bash.
4. **One-liner where shell-startup cost dominates** — a single command that takes <50 ms of actual work; nushell module load + parser warm-up would multiply runtime by an order of magnitude. Examples: a `~/.local/bin/` shim that just exec's another binary with a flag.
5. **Hard external POSIX contract** — the consumer (a system service, a vendor's hook, a CI runner) explicitly requires `#!/bin/sh` or `#!/bin/bash` and no other interpreter. *"POSIX-required because shell scripts are POSIX"* does not count — there must be a named external consumer.

If a script handles structured data (JSON, YAML, TOML, tables), composes the workflow-tooling layer, or is invoked from nushell, it is nushell — no exception, no matter how short.

## consequences

Positive:
- Bash fallback is now testable: every new `*.sh` file must justify under one of the five conditions or be rewritten in nushell.
- Closes the "POSIX-required" escape hatch from [[adr0001]] — POSIX alone is no longer sufficient.
- Reviewers have a checklist instead of taste-based arguments.

Negative:
- Some existing scripts under `scripts/ubuntu/` and `nushell/actions/` may straddle the boundary — an audit pass is needed to either justify or migrate them.
- Condition 4 (the 50 ms threshold) is a heuristic; edge cases will need judgment.
- New scripts that genuinely deserve bash now need a one-line comment naming which condition applies, which is mild friction.
