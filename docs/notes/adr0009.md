# ADR [[cat002]] [[product]]

## title
preview-d embeds every peer daemon as a direct cross-origin iframe

## context
Superseding [[adr0006]], which split embedding by frame-header behavior:
akm-graph direct cross-origin, d2-router through a same-origin `/d2embed/`
proxy that stripped `X-Frame-Options` / CSP. That split rested on a
hypothesis [[poc006]] itself flagged as never run-tested — that d2-router's
`d2 --watch` page *can* emit frame-blocking headers.

Running it (dotfiles-ars) disproved both halves. d2-router at v0.7.1 — the
same version poc006 probed — emits only `Content-Type`, `Date` and
`Transfer-Encoding`: no `X-Frame-Options`, no CSP `frame-ancestors`. The
proxy defended against nothing. Worse, it broke the feature it enabled:
`d2 --watch` serves an empty `#d2-svg-container` and pushes the SVG over a
websocket `watch.js` opens at a root-relative `/{project}/{file}/watch`,
beside root-relative `/{project}/{file}/static/*` assets. Fronting only the
document left both resolving against preview-d, which serves neither, so
the diagram rendered blank with no error anywhere. Proxying the document
alone cannot work; proxying the rest would mean carrying d2-router's whole
asset + websocket surface — what poc006 explicitly warned against.

## decision
preview-d embeds every peer daemon as a direct cross-origin `<iframe>` at
the daemon's own URL — `renderD2Embed` asks [[ft002]]'s `GET /api/resolve`
for d2-router's URL and embeds that, exactly as `renderAkmEmbed` already
did for [[ft004]]. The `/d2embed/` route, `proxyAndStripFrameHeaders`, and
the frame-header machinery are deleted. Verified end-to-end: the diagram
renders and live-reloads on file edit.

## consequences
Live preview works at all — origin-relative assets and websockets resolve
natively, which no document-only proxy could achieve. d2 now matches akm,
removing a special case and 195 lines. No proxy hop.

The cost is real: preview-d now has ZERO visibility into embedded
responses. A backend that ever does emit frame-blocking headers cannot be
embedded — no mitigation short of changing that backend, since a proxy is
not an option for websocket-driven backends anyway. This trades a
capability we never needed for one we did, and we would rediscover the
loss only when adding a frame-blocking peer.

The `resolveInRoot` path-jail [[adr0006]] cited on the proxy route no
longer guards the embed path; `handleFile` still jails `/file/<path>`
before dispatch, so validation survives upstream, but the browser now
talks to d2-router directly and d2-router's own index is the only gate on
what it will serve. It also forecloses serving preview-d to a REMOTE
browser without exposing every peer daemon's port too — fine for a local
webview, a wall if preview-d is ever remoted.
