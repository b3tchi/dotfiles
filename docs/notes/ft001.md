# Feature [[cat001]] [[product]]

## providing
A nushell module (`project`) that registers filesystem locations as
named projects in a single YAML file
(`~/.config/project/projects.yaml`), lists them, navigates to them, and
auto-cd's to the project directory when a shell starts on a WM
workspace matching a project name. Consumed by the WM project picker,
tmux project picker, and any shell session whose `PROJECT` env var (or
detected workspace name) resolves to a registered project.

## api_surface
Nushell module at [nushell/scripts/project/mod.nu](../../nushell/scripts/project/mod.nu), used via `use
scripts/project` (or autoloaded). Public commands:

- `project list` — list registered projects, mark active tmux session
  groups with `*`. Returns table `{ name, path, ssh, active }`. The
  `ssh` column is empty for local entries and holds the profile name
  for remote entries.
- `project add <name: string> [path?: string] [--ssh <profile>]` —
  register a project. `path` defaults to `$env.PWD` for local entries.
  `--ssh <profile>` marks the entry as remote: `path` is then
  interpreted on the remote host and must be supplied explicitly (no
  PWD default). Errors if name already exists, if `--ssh` is set with
  an empty profile, or if `--ssh` is set without an explicit `path`.
- `project remove <name: string>` — unregister. Completes against
  registered names.
- `project lookup <name: string>` — return the registry entry record
  for a name: `{ path }` for local entries, `{ path, ssh }` for remote
  entries, `null` for unregistered names. Does not error on missing —
  null is the documented "missing" signal. Consumed by WM-side
  dispatchers (e.g. `wm-launch-terminal`) that need to branch on the
  `ssh` field without re-parsing yaml.
- `project config` — open `~/.config/project/projects.yaml` in
  `$EDITOR`. Creates the file with `{ projects: {} }` if missing.
- `project go <name: string>` (env-modifying) — dispatch branches on
  the `ssh` field:
  - **Local** (no `ssh`): cd to project path, then outside tmux →
    `tmux-start attach 0 <name>`; inside tmux → switch-client to
    existing session group or create new one.
  - **Remote** (`ssh` set): delegates entirely to
    `tmux-to-workstation connect -g <name> --cwd <path> -- <ssh-alias>`.
    No local `cd` — the path lives on the remote host. Connection
    parameters (user, host, port) are resolved by ssh from
    `~/.ssh/config.d/<profile>`.

Exports `$env.PROJECT_CONFIG_PATH` on load. Auto-cd on shell startup:
if WM IPC is available, queries focused workspace and extracts project
name (`dotfiles_1 → dotfiles`, `3 → null`); otherwise falls back to
`$env.PROJECT`. Remote entries are silently skipped during auto-cd —
initiating an ssh connection on every shell open would be surprising
and breaks offline. Remote is opt-in via explicit `project go`.

## data_model
Single YAML file `~/.config/project/projects.yaml`:

```yaml
projects:
  <name>:
    path: <absolute path>          # required; remote path when ssh set
    ssh: <profile>                 # optional; ~/.ssh/config.d/<profile>
```

The `ssh` field is optional. When absent the entry is local (today's
shape, unchanged). When present the entry is remote: `path` is
interpreted on the remote host and `project go` delegates to
`tmux-to-workstation`. A mixed registry of local and remote entries is
valid.

Example with both shapes:

```yaml
projects:
  dotfiles:
    path: /home/jan/.dotfiles         # local entry
  myapp:
    path: /home/user/repos/myapp      # remote entry
    ssh: workstation                  # ~/.ssh/config.d/workstation
```

`ssh: null` and `ssh: ""` are normalised to absent (treated as local).
Any other non-string value for `ssh` is rejected at parse time with a
loud error. No workspace assignments, no session state, no additional
metadata. Ownership: the developer's home directory, edited via
`project add` / `project remove` / `project config`. Retention:
persistent until the user removes an entry.

## sample
Local project (default workflow — unchanged):

```nushell
cd ~/repos/my-new-project
project add my-new-project        # registers $env.PWD under the name
project list                      # see all registered, with tmux activity
project go my-new-project         # cd + tmux session switch/create
project remove my-new-project     # unregister
```

Remote project (new — path lives on the remote host):

```nushell
# Register a remote project; path is the directory on the workstation.
# SSH connection params are in ~/.ssh/config.d/workstation.
project add myapp /home/user/repos/myapp --ssh workstation

project list
# name     path                      ssh           active
# -------  ------------------------  ------------  ------
# dotfiles /home/jan/.dotfiles                     *
# myapp    /home/user/repos/myapp    workstation

project go myapp   # → tmux-to-workstation connect -g myapp --cwd /home/user/repos/myapp -- workstation
                   # opens a thin local tmux window; remote session starts at /home/user/repos/myapp
```
