# Feature [[cat001]] [[product]]

## providing
A nushell module (`project`) that registers filesystem locations as named projects in a single YAML file (`~/.config/project/projects.yaml`), lists them, navigates to them, and auto-cd's to the project directory when a shell starts on a WM workspace matching a project name. Consumed by the WM project picker, tmux project picker, and any shell session whose `PROJECT` env var (or detected workspace name) resolves to a registered project.

## api_surface
Nushell module at `nushell/scripts/project/mod.nu`, used via `use scripts/project` (or autoloaded). Public commands:

- `project list` ‚Äî list registered projects, mark active tmux session groups with `*`. Returns table `{ name, path, active }`.
- `project add <name: string> [path?: string]` ‚Äî register a project. `path` defaults to `$env.PWD`. Errors if name already exists or path missing.
- `project remove <name: string>` ‚Äî unregister. Completes against registered names.
- `project config` ‚Äî open `~/.config/project/projects.yaml` in `$EDITOR`. Creates the file with `{ projects: {} }` if missing.
- `project go <name: string>` (env-modifying) ‚Äî cd to project path, then: outside tmux ‚Üí `tmux-start attach 0 <name>`; inside tmux ‚Üí switch-client to existing session group or create new one.

Exports `$env.PROJECT_CONFIG_PATH` on load. Auto-cd on shell startup: if WM IPC is available, queries focused workspace and extracts project name (`dotfiles_1 ‚Üí dotfiles`, `3 ‚Üí null`); otherwise falls back to `$env.PROJECT`.

## data_model
Single YAML file `~/.config/project/projects.yaml`:

```yaml
projects:
  <name>:
    path: <absolute path>
```

Name-to-path only ‚Äî no workspace assignments, no session state, no metadata. Ownership: the developer's home directory, edited via `project add`/`project remove`/`project config`. Retention: persistent until the user removes an entry.

## sample
```nushell
cd ~/repos/my-new-project
project add my-new-project        # registers $env.PWD under the name
project list                       # see all registered, with tmux activity
project go my-new-project          # cd + tmux session switch/create
project remove my-new-project      # unregister
```

Full README at `nushell/scripts/project/README.md`.

## components
- `nushell/scripts/project/mod.nu` ‚Äî module source
- `nushell/scripts/project/README.md` ‚Äî capability documentation
- `~/.config/project/projects.yaml` ‚Äî runtime data store (user-managed)
- depends on `nushell/actions/wm-ipc.nu` for workspace-name auto-cd (optional ‚Äî falls back to `$env.PROJECT`)
