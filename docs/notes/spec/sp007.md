# Spec [[cat002]] [[board]]

## solves [[us008]]

## problem

The developer ([[pn001|developer]]) needs to use the native Manjaro
workstation as a primary work environment from other devices (phone,
laptop) — the full graphical desktop, not just a shell. [[us008]] frames
the capability; the constraints (trusted-network-only reachability,
connection-specific encryption, account allowlist, brute-force throttle)
live in its acceptance criteria.

This sits in [[cat002]] (display-platform). [[adr0004]] already commits
WSL2 full-desktop remoting to xrdp + xorgxrdp on llvmpipe, and [[adr0005]]
forbids d3d12 outside WSLg — both are precedent for the remoting stack but
were scoped to WSL2. This spec extends the same stack to a **native**
host, where a real GPU is present and the host has no Windows RDP-port
clash. The session-layering glue from [[ft003]] (i3-config-layering) is a
candidate consumer: the remote session needs its own overlay (modifier,
compositor backend) without disturbing the local console session.

The novel surface is **security hardening**, which no existing
[[adr0004]]/[[adr0005]] decision covers: replacing the bundled TLS key
with a connection-specific certificate and refusing legacy RDP security,
restricting reachability to LAN / OVPN at the firewall, gating logins to
an explicit account group, disabling privileged-account remote login, and
rate-limiting failed attempts. Whether that hardening posture becomes a
binding [[cat002]] decision is a retro question, not a precondition here.
