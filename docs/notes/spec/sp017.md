# Spec [[cat002]] [[board]]

## solves
[[us015]]

## problem
The four i3/quickshell dialogs — launcher (mod+d), window switcher
(mod+w), projects (mod+p), clip picker (mod+v) — are the same control
(input bar + type-to-filter + arrow-nav row list + Enter-confirm /
Esc-cancel) but share no code. Three (launcher/switcher/projects) live in
`quickshell/overlay/shell.qml`, duplicated verbatim in
`quickshell/config/Overlay.qml` (RDP single-instance host); the two copies
have already drifted (`find -L` symlink fix present in one, not the other,
leaving the RDP launcher blind to rotz-linked bins). The clip picker
(`config/ClipHistory.qml`) is a separate implementation with its own 720px
chrome, a plain substring filter, and NO fuzzy match or match-char
highlight — the other three have scored `fuzzyMatch()` + `highlightMatch()`.
This is the [[us015]] "picker-overlay-visual-parity" story widened from a
one-picker restyle to formalizing the shared combobox contract.

Formalize what the four controls have in common into one reusable
component under `quickshell/config/Common/` (alongside `Session.qml`):
- a Combo control: input bar + filtered list frame + keyboard contract
  (type filters, arrows move, Enter confirms, Esc cancels), fzf-style
  match highlight in rows
- a Fuzzy module: scored `fuzzyMatch()` + `highlightMatch()` extracted
  from Overlay.qml, single implementation
- a DialogTheme: geometry + colors (480 width, 32px input bar #152024,
  32px rows, max 8 visible, #222D31 body, #FDF6E3 text, Iosevka Nerd
  Font, NativeRendering) matching the Overlay.qml constants

Per-dialog concerns stay per-dialog (supplied to the Combo): model
source (PATH bins / WM tree / project registry / clip store), row
renderer, confirm action (exec / focus window / open project / publish
entry id), and special modes (switcher mod-held cycle; clip id-stable
selection publishing `filtered[index].row`, never positional — [[adr0010]]).

Consumers refactor onto Combo:
- launcher / switcher / projects — `config/Overlay.qml` becomes the single
  source; `overlay/shell.qml` shrinks to a thin ShellRoot wrapper loading
  it, killing the duplicate and folding in the `find -L` fix.
- ClipHistory adopts Combo — gains fuzzy filter + match highlight + parity
  chrome as a side effect. Stays hosted in the MAIN quickshell instance on
  every session type (sp014/[[ft007]] IPC design unchanged — restyle, not
  relocation).

Migration/behavior delta: clip picker filter changes substring -> fuzzy
with highlight (user-visible). Other three render identically. No ADR
overturned; [[adr0004]] dual-session and [[adr0010]] publish-not-paste
both respected. Supersession call: [[us015]] AC is superseded by the wider
combobox-formalization AC at spec-writing; [[im005]] untouched (unrelated).
The clip picker's im### (if any) and the overlay have no accepted im### —
this spec will mint the implementation card at im-write time.

## affected
- [[us015]] picker-overlay-visual-parity (widened)
- [[ft007]] shared-clipboard-history (clip picker consumer)
- [[adr0010]] publish-not-paste (respected)
- [[adr0004]] dual-session (respected)

## solution
Mint the shared control as a first-class Feature — [[ft008]]
(shared-combobox-dialog-control) — rather than folding it into a single
dialog's code. [[ft008]] provides the reusable combobox (input bar +
fuzzy-scored filter + match-char highlight + arrow-nav list +
Enter-confirm / Esc-cancel) as three QML units under
`quickshell/config/Common/` next to `Session.qml`: a `Combo` control, a
`Fuzzy` module (the scored `match` + `highlight` extracted from
[Overlay.qml](../../../quickshell/config/Overlay.qml)), and a
`DialogTheme` singleton holding the geometry + colors. Each dialog stays
the owner of its model, row renderer, and confirm action — [[ft008]] owns
only the interaction contract and chrome.

Consume [[ft008]] from all four dialogs, delivered via the [[ft003]]
i3-config-layering keybind + quickshell host layer that already wires
them:
- launcher / switcher / projects — refactor
  [Overlay.qml](../../../quickshell/config/Overlay.qml) onto the `Combo`;
  collapse the drifted duplicate
  [overlay/shell.qml](../../../quickshell/overlay/shell.qml) into a thin
  `ShellRoot` wrapper that loads the same `Overlay.qml`, folding in the
  `find -L` symlink fix as the two copies merge.
- clip picker — refactor
  [ClipHistory.qml](../../../quickshell/config/ClipHistory.qml) onto the
  same `Combo`; it gains the fuzzy filter, match highlight, and parity
  chrome as a side effect, satisfying the [[us015]] parity AC. It stays
  hosted in the MAIN quickshell instance on every session type — the
  [[ft007]] / [[adr0010]] IPC + publish-not-paste design is a restyle,
  not a relocation, and the id-stable selection contract (publish the
  caller's `row` id, never a positional index) is preserved by the
  `Combo` confirm handler passing the caller's row object back.

Trade-off taken: one refactor pass touching four dialogs now, in exchange
for a single fuzzy/theme source and the death of the `overlay/shell.qml`
↔ `Overlay.qml` drift. No [[adr0004]] / [[adr0010]] overturn — both bind
and are respected. The [[us015]] AC is widened here from a one-picker
restyle to "all four dialogs consume [[ft008]]"; its concrete parity
constants move into `DialogTheme`. The consuming `im###` is minted at
[[ft008]]-anchored spec-refinement, promoting [[ft008]] `proposed →
stable` once that im lists it.

Combo contract (the [[ft008]] api_surface as widened here — tasks quote
it, do not reinvent it): core `model` / `delegate` (receives `row`,
`isSelected`, `matchIndices`) / `onConfirm(row)` / `onCancel()`; knobs
`textOf(row)->string` (default `row.name`), `filterMode`
"fuzzy"|"external", `placeholder`, `inputVisible` (default true),
`maxVisibleRows` (default `DialogTheme.maxRows`, -1 unbounded),
`minVisibleRows` (default 0), `comboWidth` (default
`DialogTheme.width`), `emptyText`, `altConfirmEnabled` (Shift+Enter →
`confirmAlt(row)`; false → Shift+Enter behaves as Enter); imperative
`next()` / `prev()` / `confirmCurrent()` / `setIndex(i)` /
`forceFocus()`, readonly `filtered` + writable `index`, readonly
`search`. Combo is an Item, NOT a Window: each dialog keeps its own
Window chrome and binds height to Combo `implicitHeight` =
`(inputVisible ? inputHeight : 0) +
max(min(filtered.length, maxVisibleRows), minVisibleRows) * rowHeight
+ pad` — this reproduces every existing height formula (launcher
32+min(n,8)*32+8; switcher plain max(n,1)*32+8; clip floor-1 variant).

Conventions:
- Test discipline copied from `test-clip-history.sh`: Xvfb displays
  :95+, argv-logging stubs on a sandboxed PATH, isolated
  `XDG_RUNTIME_DIR`, named scenarios, negative controls that fail on
  the specific mutant (positional id, reverted `find -L`, substring
  filter).
- `Fuzzy.highlight` HTML-escapes `&`, `<`, `>` BEFORE marking matched
  chars — clip previews are arbitrary clipboard bytes rendered as
  RichText once highlighting lands; unescaped markup is an injection
  into the row renderer.
- Ids stay opaque strings end to end (dotfiles-g5b): no parseInt, no
  re-stringification; Combo hands back the row OBJECT, consumers pick
  their own field off it.
- Test suites are bash like their siblings ([[adr0002]] — UI observed
  indirectly, logic asserted via sh-visible effects).
- Singleton style follows `Session.qml` (`pragma Singleton` +
  qmldir `singleton` entries).

Anti-patterns (auto-reject in review):
- Publishing a positional index or the first filtered row instead of
  the selected row object ([[adr0010]], dotfiles-g5b class).
- Synthesizing keystrokes or inspecting focused windows ([[adr0010]]).
- Duplicating any DialogTheme constant at a call site, or leaving
  `fuzzyMatch`/`highlightMatch` bodies outside `Common/` after T6.
- Editing `overlay/shell.qml` beyond the thin wrapper after T6 — the
  drift this spec kills must not be reintroducible.
- Touching `qs-clip.sh`, `clip-set.sh`, i3 binds, or the instance
  split in `config/shell.qml` — out of scope, [[ft007]]/[[ft003]]
  surfaces are consumed as-is.

Known limitations:
- Switcher keeps its dual-field (name+ws) scoring and its 640px width
  as caller-side concerns (`filterMode: "external"`, `comboWidth`) —
  formalizing multi-field scoring inside Fuzzy is deliberately out of
  scope.
- Clip picker filter changes substring → fuzzy with highlight: the one
  user-visible behavior delta, accepted in `## solution`.
- Live visual parity on a real RDP session is a manual check at merge;
  headless asserts geometry, not rasterized pixels.
- [[ft008]] stays `proposed` through refinement; the `→ stable` flip
  lands when this spec ships (work-merge/spec-retro), matching the
  [[ft007]] precedent.

  task — extraction is additive; `git diff --name-only` shows only the
  four files above.

#### edge_cases
- Empty text with non-empty pattern → not matched.
- Pattern longer than text → not matched.
- Case folding: pattern uppercase matches lowercase text and vice
  versa (both sides lowered, indices refer to original text).
- Unicode: multibyte (UTF-16 surrogate) text — indices produced by
  `match` land on the same code units `highlight` marks; no split
  surrogate in output.
- `highlight` with empty/null indices returns escaped text unchanged.
