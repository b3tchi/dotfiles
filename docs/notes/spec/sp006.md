# Spec [[cat001]] [[board]]

## solves
[[us006|visualize-note-relations]]

## implements
[[im004|akm-graph-note-relations-viz]]

## problem
The AKM zettel graph (us / sp / im / ft / adr / cat / pn / poc plus the
product, board, and akm hub notes under docs/notes/) is only readable
file-by-file — there is no way to see the whole knowledge graph, spot
clusters, or catch dangling [[wikilinks]] visually. The `akm` CLI has
list/read/write subcommands but no graph export and no visualization
surface.

Proposed capability: a live graph-visualization daemon rendering the
full zettel graph with the GPU-accelerated cosmos library
(@cosmograph/cosmos). Approved brainstorm direction:

- Go daemon `akm-graph-d` (engine) + mandatory nushell wrapper
  `akm graph start|stop|open|status` per [[adr0003]] — all three
  conditions hold: long-running listener, websocket push on file
  change, not modellable as a nushell pipeline. Zettel parsing
  (frontmatter + wikilink extraction) lives in Go because it sits on
  the fsnotify watch-loop hot path; this is flagged explicitly against
  the [[adr0003]] scope boundary (no rewriting existing nu data work —
  none exists for graph extraction today).
- Nodes: every typed zettel + hub notes; color by type, size by
  degree, hover tooltip with id / alias / status. Labels/tooltips
  only — no preview panel in v1.
- Edges: every [[wikilink]] occurrence, deduped; dangling links render
  as ghost nodes (visual broken-ref diagnostic).
- Page + vendored cosmos.js embedded via go:embed (offline-safe, no
  CDN, no build step on target). `GET /api/graph` JSON, `WS /watch`
  live push, `GET /api/status`, `POST /api/stop`, port
  AKM_GRAPH_PORT (default 4810).

Consumers: (1) the developer browsing/auditing the knowledge graph in
a browser via the nu wrapper; (2) scripts and skills consuming the
machine-readable `GET /api/graph` JSON for link analysis — orphan and
dangling-ref detection, future `akm check` integration. No existing
implementation carries graph-viz logic ad-hoc today, so there are no
migration targets; [[im003]] (neovim-d2-diagramming) is the closest
consumer-pattern precedent (daemon + browser preview).

Dedup check: no existing feature covers this — [[ft001]] (project
registry), [[ft002]] (d2-preview-router), [[ft003]] (i3 config
layering) were surveyed; [[ft002]] is the architectural precedent
(single-port Go daemon, lazy work, nu wrapper, pidfile/log under
~/.cache) and its conventions should be mirrored. Constraints inherit
from [[cat001]] (workflow-tooling): [[adr0001]] nushell-first user
surface, [[adr0002]] bash boundary untouched, [[adr0003]] Go daemon
gate + nu wrapper mandate + rotz install guarding `command -v go`.

A new ft### (akm-graph daemon) is minted at spec-writing time, not
here.

## solution
Solve [[us006|visualize-note-relations]] by minting and building
[[ft004|akm-graph]]: a Go daemon `akm-graph-d` mirroring [[ft002]]
conventions — single port (`AKM_GRAPH_PORT`, default 4810), pidfile +
log under `~/.cache/akm-graph/`, control API, rotz `go build` install
guarded by `command -v go`. The daemon walks `docs/notes/**.md` +
`docs/board.md`, parses frontmatter + `[[wikilinks]]` in Go (the
parse sits on the fsnotify watch-loop hot path; explicitly flagged
against the [[adr0003]] scope boundary — no existing nu graph
extraction is being rewritten), serves a go:embed viewer page with
vendored cosmos.js (`@cosmograph/cosmos`, offline, no CDN), exposes
`GET /api/graph` JSON for scripts/skills, and pushes rebuilt graphs
over `WS /watch` on file change. User surface is the nushell wrapper
`akm graph start|stop|open|status` per [[adr0001]] + [[adr0003]];
the [[adr0002]] bash boundary is untouched. Taxonomy: [[cat001]]
workflow-tooling.

Trade-offs taken: wikilink/frontmatter parsing in Go duplicates what
nushell could do offline (accepted — hot path, no prior nu
implementation); vendoring cosmos.js pins its version (accepted —
offline-safe beats freshness for a local tool).

Confidence gate: the single unproven assumption — that
`@cosmograph/cosmos` bundles to a single standalone browser file
suitable for vendoring via go:embed — is VALIDATED by [[poc003]]:
pin 1.6.1, one 830 KB self-contained ESM bundle (bun/esbuild at
vendor time), rendered 10 nodes offline in headless Chrome. Avoid
the 2.0.0-beta line until stable.

## plan
Implements [[im004]] consuming [[ft004]]; binding decisions
[[adr0001]] (nu user surface), [[adr0002]] (untouched), [[adr0003]]

File tree:

```
akm-graph/
  dot.yaml            # rotz install: guarded go build →
                      #   ~/.local/bin/akm-graph-d
  go.mod              # module akmgraph; deps: fsnotify,
                      #   gorilla/websocket, yaml.v3
  main.go             # env config, wiring, graceful shutdown
  parser.go           # notes walk, frontmatter, wikilink extraction
  graph.go            # node/link build, dedup, ghosts, degree
  server.go           # HTTP mux, go:embed static, /api/*
  watch.go            # fsnotify recursive watch, debounce, ws push
  *_test.go           # table tests per file
  fixtures/           # crafted zettel tree for parser/server tests
  static/
    index.html        # viewer shell
    app.js            # cosmos init, color/size/tooltip, ws client
    cosmos-bundle.js  # vendored @cosmograph/cosmos 1.6.1 — built
                      #   once via `bun build` per [[poc003]], committed
nushell/actions/akm-graph   # wrapper: start|stop|status|open
nushell/dot.yaml            # + link actions/akm-graph → ~/.local/bin
claude/akm/akm              # + `akm graph` passthrough subcommand
```

Conventions:

- Mirror [d2/router](../../d2/router) throughout: flat `package
  main`, table-driven tests, `net/http/httptest`, env-knob config,
  pidfile + log under `~/.cache/akm-graph/`.
- Nu wrapper mirrors
  [nushell/actions/d2-router](../../nushell/actions/d2-router):
  port-probe → pidfile → detached launch; stale-pidfile cleanup.
- Graph JSON shape is the [[ft004]] api_surface verbatim:
  `{nodes: [{id, type, status, alias, degree, ghost}],
  links: [{source, target}]}`.
- Node scope: every `*.md` under `docs/notes/**` plus hub files
  `docs/board.md`, `docs/product.md`. Type from filename prefix
  (us/sp/im/ft/adr/cat/pn/poc), hubs typed `hub`, anything else
  typed `note`.

Anti-patterns (auto-reject in review):

- Fetching cosmos or any asset from a CDN at runtime — bundle is
  vendored, page must work offline.
- `panic`/`log.Fatal` inside request handlers — errors return 500 +
  log line.
- Mutating the served graph in place — rebuild into a fresh value,
  swap under mutex (or atomic pointer); requests never see a
  half-built graph.
- Treating fsnotify as recursive — it is not; subdirs (`spec/`,
  `lab/`, `daily/`, future) must be added explicitly and newly
  created dirs watched on CREATE.
- Ignoring editor atomic-rename saves — vim/nvim write via rename;
  watch must handle RENAME/CREATE, not only WRITE.
- Rebuilding once per fs event — save bursts must debounce
  (~300 ms) into one rebuild.
- Blocking the broadcast loop on a slow ws client — drop clients
  whose send buffer is full.

Known limitations (accepted for v1):

- Wikilinks extracted by regex; links inside code fences count as
  edges. Acceptable noise for a viz tool.
- Duplicate aliases across zettels are not validated here — the akm
  CLI owns that.
- No auth on the HTTP port — localhost-only tool, same stance as
  [[ft002]].
- Not added to meta-packages; install is explicit
  `rotz install akm-graph`.

## tasks

### Task 1: zettel parser + graph builder

#### type
feature

#### effort
6h

#### depends
none

#### files_touched
- akm-graph/go.mod
- akm-graph/parser.go
- akm-graph/graph.go
- akm-graph/parser_test.go
- akm-graph/graph_test.go
- akm-graph/fixtures/**

#### success_criteria
- Walking the fixture tree yields one node per `*.md` under
  `docs/notes/**` plus the two hub files, with `type`, `status`,
  `alias` (first frontmatter alias), and computed `degree`
  ([[us006]] AC: every note appears as a node).
- Every `[[wikilink]]` occurrence yields exactly one deduped
  source→target link; `[[id|label]]` and `[[id#section]]` forms
  resolve to `id` ([[us006]] AC: every wikilink appears as an edge).
- A wikilink whose target has no file yields a ghost node
  (`ghost: true`) ([[us006]] AC: dangling links visible).
- Graph for the full fixture tree matches a golden JSON file
  byte-for-byte after stable sort.

#### edge_cases
- Note without frontmatter or without `aliases` → node with empty
  alias, type from prefix, no crash.
- Self-link `[[us001]]` inside `us001.md` → dropped.
- Duplicate links between same pair → one edge.
- Unicode in aliases and body.
- Empty notes dir → empty nodes/links arrays, valid JSON.
- Files with `.md` in subdirs `spec/`, `lab/`, `daily/` typed by
  prefix (sp/poc) or `note`.

#### test_plan
- Table tests on `extractWikilinks`: plain, piped label, section
  anchor, multiple per line, inside code fence (documented as
  counted), malformed `[[` without close.
- Table tests on frontmatter parse: full, missing aliases, missing
  status, no frontmatter at all.
- Golden-file test on full fixture tree (nodes + links sorted);
  regenerating golden requires explicit `-update` flag.
- Degree + ghost assertions on a crafted dangling-link fixture.

### Task 3: HTTP server, embed, control API

#### type
feature

#### depends
Task 1, Task 2

#### files_touched
- akm-graph/main.go
- akm-graph/server.go

#### success_criteria
- `GET /` serves the embedded viewer (go:embed of `static/`);
  works from a binary run in an empty cwd.
- `GET /api/graph` returns the current graph JSON in the [[ft004]]
  shape; `GET /api/status` returns pid, root, node/link counts,
  last rebuild time; `POST /api/stop` shuts down gracefully
  ([[us006]] AC: machine-readable data).
- Port from `AKM_GRAPH_PORT` (default 4810); notes root from
  `--root` flag or `AKM_GRAPH_ROOT` env, no hardcoded path.
- Wrong method on any `/api/*` → 405 with `Allow` header (mirrors
  [[ft002]] behavior).

#### edge_cases
- Port already bound → clear error on stderr, nonzero exit (no
  silent retry loop).
- Concurrent `/api/graph` requests during a rebuild → consistent
  snapshot (lock/atomic swap from plan anti-patterns).
- Root path missing/not a dir → startup error naming the path.
- `GET /api/stop` (wrong method) → 405, daemon stays up.

#### test_plan
- httptest suite: graph JSON golden against fixture root, status
  field presence, stop handler triggers shutdown hook (observable
  via context cancel in test), 405 matrix over endpoints × methods.
- Embed test: binary serves index.html + cosmos-bundle.js with
  correct content-type from a temp cwd.

#### success_criteria

#### edge_cases

  serving last graph (no crash).
- WS upgrade on `/watch` with wrong method/no upgrade header →
  400/405, not a hang.

#### success_criteria

- `akm-graph-d` binary missing → actionable message naming
  `rotz install akm-graph`.
- `AKM_GRAPH_PORT` override respected by probe, open URL, and
  stop.
- `open` with daemon down → starts it first (mirror d2-router
  `open` behavior).

#### test_plan
