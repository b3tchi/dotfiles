# Feature [[cat001]] [[product]]

## providing
A single-port HTTP daemon (`d2-router-d`, Go) that owns live d2
previews across all registered projects: lazy-spawns one `d2 --watch`
child per requested diagram, reverse-proxies HTTP + websocket under
flat `/{project}/{filename}` routes, serves an index of every `*.d2`
file across [[ft001]] registry projects, and exposes a control API
(status / force reload / restart / stop). Consumed by the nvim d2
integration and ad-hoc browser/CLI use.

## api_surface
HTTP on `localhost:$D2_ROUTER_PORT` (default 4800):

- `GET /` — index page: registered projects × their `*.d2` files,
  child status, reload/stop controls.
- `GET /{project}/{filename}` — diagram page; lazy-spawns child
  `d2 --watch --browser 0` on a router-assigned port; proxies with
  two body rewrites (`/static/*` asset prefix, watch.js ws path).
- `WS /{project}/{filename}/watch` — upgrade-proxy to the child's
  `/watch`; d2's native live reload preserved.
- `GET /api/status` — JSON: children `{project, file, port, pid,
  clients, lastCompile, lastError}`.
- `POST /api/reload/{project}/{filename}` — force reload: touch the
  watched file → child recompiles + broadcasts.
- `POST /api/restart/{project}/{filename}` — kill + respawn child
  (browser ws auto-reconnects).
- `POST /api/stop/{project}/{filename}`, `POST /api/stop-all`.
- `GET /api/resolve?path=<abs>` — map an absolute file path to its
  route: path is canonicalized (`filepath.Clean`) before the
  longest registry-path prefix match (segment-aware) →
  `{project, file, url}`; 404 outside all local projects, 400 on
  relative paths. Consumed by the nvim integration (no yaml parsing
  in lua).
- Wrong method on any `/api/*` endpoint → 405 + `Allow` header.
- `..` traversal in proxy routes → 400/404 (decoded, pre-clean
  segment check on both `URL.Path` and `RawPath`); never forwarded
  to a child.

Env knobs (all daemon-side): `D2_ROUTER_PORT` (4800),
`D2_ROUTER_CHILD_PORT_BASE` (4801), `D2_ROUTER_IDLE` (30m),
`D2_ROUTER_D2_BIN` (`d2`), `D2_ROUTER_REGISTRY` (registry path).

Nu wrapper `d2-router` ([[adr0001]]):
`d2-router start|stop|status|open|reload` — idempotent daemon
launcher + thin API client. `start` probes `/api/status`, keeps a
pidfile at `~/.cache/d2-router/pid`, cleans stale pidfiles, and
launches detached with logs at `~/.cache/d2-router/log` ([[adr0002]]
cond 4).

## data_model
No persistent store. In-process runtime state only: routing table
built from [[ft001]] `projects.yaml` (project → path, refreshed per
index request; non-empty `ssh` entries excluded), children map keyed
`{project}/{filename}` → `{pid, port, path, clients, lastError,
lastActive}`. The status field `lastCompile` is sourced from
`lastActive` (last proxy/ws activity) — d2 exposes no true compile
timestamp. Filename is the basename (flat routing); duplicate
basenames within one project: first lexical wins, index page flags
the collision. Idle reaper kills children with zero ws clients past
`$D2_ROUTER_IDLE`.

## sample
```nushell
d2-router start                       # launch daemon (idempotent)
d2-router open dotfiles/network.d2    # → http://localhost:4800/dotfiles/network.d2
d2-router status                      # table of running children
d2-router reload dotfiles/network.d2  # force re-render
```

nvim: `<leader>cp` opens the routed URL (daemon auto-started),
`<leader>cP` force-reloads the current buffer's diagram.

## components
- d2/router/main.go — env config, server wiring, graceful shutdown
- d2/router/registry.go — [[ft001]] registry loader, `*.d2` walk,
  route table
- d2/router/children.go — child lifecycle: spawn / touch-reload /
  restart / reaper / crash tracking
- d2/router/proxy.go — prefix routes, body rewrites, ws upgrade-proxy,
  traversal guard
- d2/router/api.go — `/api/*` control endpoints
- d2/router/index.go — HTML index + fetch action buttons
- d2/router/*_test.go + fixtures/ — 62-test suite (fake-child helper
  pattern; fixtures probed from live d2 v0.7.1)
- nushell/actions/d2-router (+ d2-router-test) — launcher + API client;
  linked via nushell/dot.yaml to `~/.local/bin/d2-router`
- d2/dot.yaml — rotz install: `go build` → `~/.local/bin/d2-router-d`
- nvim/plugins/lang-d2.lua — migrated consumer (router flow with
  legacy per-buffer watch kept as named fallback)
