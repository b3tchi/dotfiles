# Story [[cat002]] [[product]]

## role
[[pn001|i3 desktop user]] (native and RDP sessions)

## want
the clipboard-history picker (mod+Shift+v) to look and feel like the
launcher (mod+d) and projects (mod+p) overlay dialogs — same width,
row height, input-bar chrome, fonts and colors

## because
the three pickers form one interaction family; the clip picker's
divergent 720px geometry and chrome make it feel foreign and
inconsistent with the muscle-memory dialogs

## acceptance_criteria
- picker window 480 wide, 32px input bar (#152024), 32px rows, max 8
  visible rows, #222D31 body, #FDF6E3 text, NativeRendering — matching
  Overlay.qml dialog constants
- keyboard behavior unchanged (filter, arrows, Enter publishes, Esc)
- ClipHistory stays hosted in the MAIN quickshell instance (RDP
  single-instance IPC design unchanged — restyle, not relocation)
- test-clip-history.sh suites stay green
