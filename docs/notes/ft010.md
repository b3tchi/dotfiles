# Feature [[cat002]] [[cat001]] [[product]]

## providing
The desktop-notification stack for all concurrent graphical sessions:
one user-scoped `quickshell -p notif` daemon owns
`org.freedesktop.Notifications` on the user session bus ([[adr0012]]),
appends every notification to a persistent age-capped history store
([[adr0013]]), and broadcasts live state (count / last text / seq /
critical) through an atomic state file that every bar follows with
`tail -F`. Bars are read-only consumers; a history browser recalls and
dismisses past notifications.

Solves the user-bus singleton collision: both live sessions share
`/run/user/1000/bus`, so when each per-display bar hosted its own
`NotificationServer` the first owner won and the other session's
notifications silently vanished ([[adr0004]] dual-session; fixed by
[[sp019]]). The owner/consumer split is the [[ft007]] /
`qs-stats-daemon` shape applied to notifications.

SCOPE: ticker + bell + browser; no popup windows, no action-invoke
(`actionsSupported` false), no clipboard interaction — the browser is
view + dismiss only, preserving the [[adr0010]] posture. History is
plaintext on persistent disk, age-capped ~2 days — the deliberate
departure from clipboard memory-only, recorded in [[adr0013]].

## api_surface
Keybind for the human, plus a three-verb shell contract
(`quickshell/qs-notif.sh`):

    $mod+Shift+n            toggle the browser on the active session
      type                  filter the list (newest-first kept)
      Enter                 dismiss the selected entry, stay open
      Del                   same, only while the filter is empty
      Esc                   close

    qs-notif.sh toggle      open/close the browser on the derived
                            session (live-instance derivation, the
                            qs-clip mechanism; QS_NOTIF_DISPLAY
                            forces)
    qs-notif.sh list        history as "<id>\t<preview>" lines,
                            newest first — deterministic across calls
                            over an unchanged store
    qs-notif.sh dismiss <id|latest>
                            remove the entry from history AND dismiss
                            the live notification if still tracked;
                            routed over the daemon FIFO; exits 1
                            (bounded, ≤2 s) when no daemon is alive

Daemon-internal backend (`quickshell/qs-notif-store.sh`, called by
the daemon ONLY — single writer):

    qs-notif-store.sh append      entry from stdin body + metadata
                                  args; prunes past QS_NOTIF_MAX_AGE
    qs-notif-store.sh dismiss <id|latest>
    qs-notif-store.sh state ...   atomic live-state rewrite

Ids are opaque: equality, existence, and sort only — never parsed or
arithmetic'd on (the dotfiles-g5b bug class).

## data_model
The store contract (defined in `qs-notif-store.sh`'s header; every
client is written against it):

    store dir   ${XDG_STATE_HOME:-$HOME/.local/state}/qs-notif/  0700
    entry       NNNNNN.notif — six-digit zero-padded monotonic seq
                  line 1  <epoch>\t<urgency>\t<app>
                  line 2  summary (control whitespace folded)
                  line 3+ raw body bytes
    id          the filename; opaque to consumers
    ordering    lexicographic filename order IS arrival order
    retention   entries older than QS_NOTIF_MAX_AGE (default 172800 s
                ≈ 2 days) pruned on every write
    work files  *.tmp + dotfiles; consumers glob ??????.notif only

    state file  $XDG_RUNTIME_DIR/qs-notif.state (QS_NOTIF_FILE) —
                lines `count N`, `critical 0|1`, `seq N`,
                `last <epoch>\t<text>`; atomic tmp+rename
    FIFO        $XDG_RUNTIME_DIR/qs-notif.cmd — `dismiss <id|latest>`

Entries appear atomically (`.tmp` + `ln`; collision = retry, never a
lost entry). `count` is the daemon's live TRACKED total (bell
semantics; resets on daemon restart); the store is history and
persists across restarts and logouts.

## sample
Notify from one session, see it everywhere, recall it later:

    notify-send -u critical "disk almost full" "94% used on /"
    # every live bar: ticker scrolls the text, bell count +1, red

    $mod+Shift+n            # browser opens, newest first
    <type to filter>
    Enter                   # entry dismissed, list refreshes, stays open
    Esc

    qs-notif.sh list        # 000012.notif<TAB>10:41  disk almost full — 94% used on /
    qs-notif.sh dismiss latest; echo $?    # 0

## components
- [quickshell/notif/shell.qml](../../quickshell/notif/shell.qml) —
  daemon profile: the single NotificationServer + FIFO reader
- [quickshell/qs-notif-store.sh](../../quickshell/qs-notif-store.sh)
  — backend: defines the store contract, atomic writes, age prune,
  state rewrite
- [quickshell/qs-notif.sh](../../quickshell/qs-notif.sh) — the whole
  non-UI outside layer in sh ([[adr0002]])
- [quickshell/qs-start.sh](../../quickshell/qs-start.sh) —
  user-level flock singleton launch, hash-check restart, session
  kill-sweep exclusion
- [quickshell/config/NotifHistory.qml](../../quickshell/config/NotifHistory.qml)
  — presentation only; talks to the store solely through
  `qs-notif.sh`
- [quickshell/config/Bar.qml](../../quickshell/config/Bar.qml) —
  read-only consumer: state-file tail drives ticker/bell/count
- [i3/config](../../i3/config) +
  [i3/config.common](../../i3/config.common) — `$mod+Shift+n` bind +
  `qs-notif` for_window rule ([[ft003]])
- Test suites: `quickshell/test-notif-store.sh`,
  `quickshell/test-notif-daemon.sh`,
  `quickshell/test-notif-history.sh` — headless, isolated bus/XDG
  dirs, mutation-tested
