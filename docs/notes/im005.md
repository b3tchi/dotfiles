# Implementation [[cat002]] [[product]]

## solves
[[us007|work-remotely-on-development-machine]]

## approach
Run a headless GNOME Wayland session on WSL and serve it over RDP so the developer reaches the full dev desktop from another device with no monitor and no local login. `gnome-shell --headless --virtual-monitor` renders to a vgem-provided DRM render node (WSL2 ships no /dev/dri); per [[adr0005]] it stays on software llvmpipe — d3d12 is forbidden outside WSLg — and `gnome-remote-desktop-daemon --headless` serves RDP on port 3390. The session is a systemd user unit started via `loginctl enable-linger`, so it survives reboot without an interactive login, and vgem is persisted via /etc/modules-load.d/vgem.conf. It coexists with the i3+xrdp stack ([[adr0004]]) on a separate port (3390 vs xrdp 3391), adding a Wayland-native remote path alongside the X11 one. De-risked end-to-end in [[poc004]]; key trade-off: software RFX encode only (no GPU hwaccel on WSL), so it is CPU-bound and best for light interactive use.

## features
none — bespoke glue, consumes no reusable [[ft###]]; elevate to a feature only if a second consumer appears

## data_model
none — the session owns no persistent state; grd credentials + TLS cert live under the user's runtime dir (`~/.local/share/gnome-remote-desktop/`), never committed

## api_surface
RDP service on `localhost:3390` (NLA/credentials auth against a grd-owned credential, not PAM). No code API added; operational surface is the systemd user unit `gnome-rdp-session.service` plus `grdctl --headless` config (TLS cert, port 3390, credentials).

## components
- [gnome-rdp/dot.yaml](../../gnome-rdp/dot.yaml) — rotz app: installs packages, persists vgem, enables linger, configures grd (TLS + port 3390), enables the unit
- [gnome-rdp/wsl/gnome-rdp-session](../../gnome-rdp/wsl/gnome-rdp-session) — launcher: headless gnome-shell + grd daemon with software-GL env (adr0005)
- [gnome-rdp/wsl/gnome-rdp-session.service](../../gnome-rdp/wsl/gnome-rdp-session.service) — systemd user unit, autostart via linger
- [meta-wsl-gnome/dot.yaml](../../meta-wsl-gnome/dot.yaml) — meta: layers the full `gnome` group + Forge (`gnome-shell-extension-forge`, i3-like tree tiling) on this app
