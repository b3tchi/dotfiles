# Story [[product]]

## role
[[pn001|i3 desktop user]] (RDP sessions)

## want
the clip picker (mod+v) to close instantly on Esc over RDP, the same as
the launcher/switcher/projects dialogs

## because
during the [[sp017]] demo a 1-2s delay was seen between Esc and the clip
picker disappearing on the xrdp :10 session; it did not reproduce
reliably afterward, so the [[sp017]] retro could not root-cause it. The
config.common qs-clip float-rule gap found during investigation was
fixed (that rule was genuinely missing on the RDP base), but the
residual lag is most likely a cold-start focus race on the first
mod+v — the window maps before X delivers keyboard focus, so the first
Esc is lost until the focus-loss guard fires. Captured here so the next
cycle can instrument live focus timing if it returns.

## acceptance_criteria
- reproduce deterministically: measure Esc->hide latency on the first
  mod+v after a fresh qs-start.sh over RDP vs warm opens
- if confirmed a focus race, the picker forces X keyboard focus on map
  (not only after qs-clip.sh list returns) so the first Esc always lands
- launcher/switcher/projects Esc latency unchanged; ClipHistory hosting
  ([[adr0004]]) and publish contract ([[adr0010]]) untouched
- test-clip-history.sh stays green
