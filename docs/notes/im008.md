# Implementation [[cat002]] [[cat001]] [[product]]

## solves
none — fix+feature spec
[[sp019|notification-service-and-history-browser]]; the
`org.freedesktop.Notifications` first-owner-wins collision between the
`:0` and `:10` bars ([[adr0004]]) was captured directly at spec stage,
with the history-browser scope promoted to [[ft010]]. No source
`us###`.

## approach
Split the notification stack into one owner + many read-only
consumers — the shape [[ft007]] and the `qs-stats-daemon` singleton
already prove on this repo, recorded as [[adr0012]].

**Owner.** A user-scoped `quickshell -p notif` daemon
(`quickshell/notif/shell.qml`) hosts the single `NotificationServer`
and owns the user-bus name. It is launched from `qs-start.sh` behind a
USER-LEVEL flock + pgrep guard (the `qs-stats-daemon` mould, including
hash-check restart-on-change), not the per-display `qs_kill_session`
scoping — that axis keeps two *bars* apart and is wrong for a bus
singleton; the session kill sweep explicitly spares the daemon's
cmdline. On each notification the daemon appends an entry to a
persistent file-per-entry store under
`${XDG_STATE_HOME:-$HOME/.local/state}/qs-notif/` (seq + timestamp,
the `clip-store.sh` shape; age-capped ~2 days, pruned on write —
the deliberate memory-only departure recorded in [[adr0013]]) and
atomically rewrites a live-state file under `$XDG_RUNTIME_DIR`
(count / critical / seq / last). Dismiss commands arrive over a FIFO;
only the daemon ever mutates store or state (single-writer). All
store/glue logic lives in `#!/bin/sh` scripts per [[adr0002]].

**Consumers.** `NotificationServer` and every notif-state property are
ripped out of `config/shell.qml` and `Bar.qml`; ticker, bell, and
count are driven by `tail -F` on the live-state file — the same
source model `Bar.qml` already uses for qs-stats. Every session
renders identical notifications because they read one file.

**Browser.** `config/NotifHistory.qml` instantiates the shared
[[ft008]] `Combo` (`filterMode: external`, newest-first — the clip
picker shape from [[im006]]), hosted in the MAIN instance beside
`ClipHistory`, reached over `qs-notif.sh toggle/list/dismiss` and the
[[ft003]] keybind layer (`$mod+Shift+n`; the name-workspace bind
relocates to `$mod+Shift+o`). View + dismiss only — Enter/Del
dismisses through the FIFO, no clipboard copy, no action-invoke
(`actionsSupported` stays false), preserving the [[adr0010]] posture.

  shape, script split, session derivation); not consumed, not touched

## data_model
Owned by [[ft010]]: persistent entry store
`${XDG_STATE_HOME:-$HOME/.local/state}/qs-notif/NNNNNN.notif`
(header `<epoch>\t<urgency>\t<app>`, summary line, raw body), atomic
tmp+`ln` writes, age-cap prune on write; live-state file
`$XDG_RUNTIME_DIR/qs-notif.state` (`count`/`critical`/`seq`/`last`),
atomic tmp+rename; command FIFO `$XDG_RUNTIME_DIR/qs-notif.cmd`.
Live tracked count is daemon memory (resets on restart); the store is
history and survives.

## api_surface
`qs-notif.sh toggle | list | dismiss <id|latest>` plus the
`$mod+Shift+n` keybind — the full contract is [[ft010]]'s
api_surface. No bar exposes any notification surface anymore.
