# Feature [[cat002]] [[product]]

## providing
Region screen capture for X11 sessions: the user selects a rectangle on the
live desktop and gets a PNG on disk plus that file's PATH on the clipboard
(text, like a tmux copy — not the image itself, so it pastes into an editor,
a chat box or a shell as a path). Consumed by the i3 keybind layer on every
X11 platform that runs the shared base config: native :0, xrdp :10 (WSL and
desktop), proot/Termux:X11.

Compositor-free by construction — the selector never blends with the screen
behind it, so it needs no compositor and works identically with or without
picom. See [[poc008]].

SCOPE: region only. The `i3-scrot` binds (`Print` whole, `$mod+Print` window,
`$mod+Shift+Print` select) are an older, separate capture path in `i3/config`
and are deliberately NOT part of this feature. `ksnip` is not a capture path
either — annotation-only (`ksnip -e <file>`), a consumer of this output.

## api_surface
Keybind-driven, no programmatic callers:

    $mod+Shift+s          enter the i3 "screenshot" mode + launch the overlay
      drag                select a region -> capture, save, copy path, exit
      click,click         2-tap corners (touch-friendly over RDP) -> as above
      w                   whole screen -> save, copy path, exit
      Esc | right-click   cancel, capture nothing

Output contract:

    file      ~/Pictures/screenshots/shot_<YYYYMMDD-HHMMSS>.png
    clipboard the absolute path of that file, as text (CLIPBOARD selection)
    exit      i3 returns to mode "default"; nothing left running

## data_model
No persistent state. Two transient artefacts:

    ~/Pictures/screenshots/shot_<ts>.png   the saved capture (permanent)
    $XDG_RUNTIME_DIR/qs-shot-src.$QS_SID   session-scoped pointer, removed on exit

`$QS_SID` is the filename-safe display id from `quickshell/qs-session.sh`
(":0" -> "_0"), so concurrent :0 and :10 sessions never share state — the
collision class in dotfiles-8xt.

## sample
Select a region and paste the path into a note:

    $mod+Shift+s          # crosshair over the LIVE desktop, bar shows the mode
    <drag a box>          # outline in the mode colour #cb4b16
    # -> ~/Pictures/screenshots/shot_20260715-142031.png saved
    # -> that path is now on the clipboard

    ksnip -e "$(xclip -selection clipboard -o)"   # annotate it, optionally

## components
- `quickshell/qs-screenshot.sh` — launcher: resolves the session, execs the
  overlay. Forces `GDK_BACKEND=x11` (WSLg leaves a `wayland-0` socket that GTK
  auto-connects to even with `WAYLAND_DISPLAY` unset).
- `quickshell/qs-region.py` — the overlay: GDK window whose X Shape BOUNDING
  region is cut to the rubber-band outline, seat grab for input, captures
  after release. Same stack and technique as `quickshell/qs-focus-border.py`.
- `quickshell/qs-shot-action.sh` — the i3 mode key actions (`w`, `Esc`); a
  shaped overlay under a grab is not an i3-managed window, so mode keys stay
  i3 bindings.
- `i3/config` — the `mode "screenshot"` block + `$mod+Shift+s` bind. Base, not
  a platform overlay: every X11 platform gets it (dotfiles-13q).
- `quickshell/config/Bar.qml` — `modeHints("screenshot")` renders the hint
  strip; it carries the mode signal, which is why no dim is drawn.
- `i3/picom.conf` — `focus-exclude` + `unredir-if-possible-exclude` must list
  the overlay by name for native :0 (picom matches by name; the overlay never
  takes focus by design).
- `quickshell/test-blink-measure.sh` — 60fps luma harness that proves the
  no-blink property.

PREDECESSOR: a frozen-PNG quickshell/QML overlay (`quickshell/region/`) ships
today and provides the same api_surface with a 0.35 dim, at the cost of a
67-150ms black blink per open. Replaced by [[sp012]]; see [[poc009]] for why
keeping it in QML cannot work.
