# Spec [[cat001]] [[cat002]] [[board]]

## problem

The [[ft005]] file-preview daemon shipped without its display half: the
`/preview<N>` live window is opened by hand in a browser today, while the
feature card already names a "webview client host: Wry (WebKitGTK)"
component that does not exist in the repo. [[poc007]] benchmarked Servo,
Wry and QtWebEngine and decided Wry/WebKitGTK (lightest RAM — 196 MiB
first window / ~110 MiB per added window, 1.4 MB binary, engine
preinstalled as webkit2gtk-4.1, chrome-less); the only artifact is a
throwaway harness generated inside `bench/webview/bench.sh`. This spec
builds the production client host and wires its lifecycle into the
[[adr0001]] wrapper surface.

**Entry type.** Feature-side extension of [[ft005]] — no backing `us###`
(the base feature entered as sp008, a feature-add spec). No supersession:
ft005 gets a body refresh at spec-retro (the "webview client host"
component becomes a real code path; sp008's "browser --app window is the
portable surface" note is already overtaken by [[poc007]]).

**Shape (approved at idea stage).** One wry process per window: a new
Rust crate `preview-webview/` builds `preview-wv <N> [--port P]`, a
chrome-less wry/tao window (WM_CLASS `preview-wv`, title `preview <N>`,
`GDK_BACKEND=x11`) loading `http://127.0.0.1:$PREVIEW_PORT/preview<N>`,
inheriting the caller's `DISPLAY`/`XAUTHORITY`. The CLI wrapper
(`nushell/actions/preview`) is the control surface consumers like nvim
use: new `preview window [N]` verb — idempotent spawn via per-window
pidfile `~/.cache/preview/wv-<N>.pid` with stale-pid cleanup, loud error
when the binary is missing or the daemon is down; `preview window [N]
--close` closes one; `preview stop` additionally closes all windows.
`preview start` stays daemon-only. `nvim/plugins/preview.lua`
`:PreviewStart` spawns its registered slot's window after register. i3 +
sway gain `for_window [class="preview-wv"] floating enable` rules.
Install mirrors the Go daemon recipe: `preview-webview/dot.yaml` runs a
cargo-guarded `cargo build --release` into `~/.local/bin/preview-wv`.

**Deferred.** Explicit `--display`/`--xauth` targeting flags (poc007
cross-display mixing) — first slice inherits the environment. Browser
fallback when the binary is absent — loud error instead.

**ADR flag.** [[adr0003]] mandates Go for compiled helpers; this client
is Rust because the wry/WebKitGTK binding is Rust-only. QML/WebKit was
ruled out (QtWebKit removed in Qt 5.6, no Qt6 port; QtWebView on Linux
falls back to QtWebEngine/Chromium — the heaviest poc007 option). A new
`adr####` recording the "Rust when the ecosystem lib demands it"
exception is minted at spec-retro, not now.

## solution

Build the display half of [[ft005]] as a **one-process-per-window Wry
host** ([[poc007]] decision), controlled exclusively through the
[[adr0001]] nushell wrapper — no consumer touches the binary directly
(the [[adr0003]] engine-not-interface stance, with the flagged Rust
exception since wry/WebKitGTK binding is Rust-only).

Three cooperating pieces:

1. **`preview-webview/` Rust crate → `preview-wv <N> [--port P]`.** A
   ~60-line wry/tao app: chrome-less window, WM_CLASS `preview-wv`,
   title `preview <N>`, `GDK_BACKEND=x11`, loads
   `http://127.0.0.1:<port>/preview<N>` and exits when the window
   closes. Inherits `DISPLAY`/`XAUTHORITY` from the caller, so windows
   land on whatever session spawned them (native :0, xrdp :10, WSLg) —
   consistent with the [[cat002]] per-display session model. One
   process per window keeps crash domains isolated; [[poc007]] measured
   this exact shape (~110 MiB marginal PSS per added window, shared
   engine `.so`).

2. **Wrapper lifecycle verbs** on `nushell/actions/preview`
   ([[cat001]]): `preview window [N]` — idempotent spawn guarded by
   per-window pidfile `~/.cache/preview/wv-<N>.pid` (alive → no-op,
   stale → clean + respawn — the same probe/pidfile discipline the
   daemon's `start` already uses); `--close` SIGTERMs one window;
   `preview stop` closes all windows before stopping the daemon; loud
   errors when `preview-wv` is missing ("rotz install preview-webview")
   or the daemon is down. `preview start` stays daemon-only —
   [[ft005]]'s sample line claiming start opens window 1 is corrected
   at spec-retro.

3. **Consumers + platform glue.** `nvim/plugins/preview.lua`
   `:PreviewStart` spawns its registered slot's window after
   `register` returns — each nvim instance owns one window (the
   [[ft005]] slot model). i3 + sway configs gain
   `for_window [class="preview-wv"] floating enable` rules via the
   [[ft003]] config-layering mechanism. Install mirrors the Go daemon
   recipe: `preview-webview/dot.yaml` with a cargo-guarded
   `cargo build --release` → `~/.local/bin/preview-wv`.

Trade-offs taken: per-process windows over a multi-window IPC daemon
(RAM parity per [[poc007]], far less coupling); loud-error over browser
fallback (predictable surface); environment-inherited display targeting
over `--display` flags (deferred, poc007 proved the mechanism when
needed). Feasibility is already proven — [[poc007]]'s harness is this
exact binary shape rendering the real viewer on both hardware :0 and
xrdp :10, so no further PoC gate is needed.

## plan

**File tree** (new crate `preview-webview/`; touched integration files
under `nushell/`, `nvim/`, `i3/`, `sway/`):

```
preview-webview/
  Cargo.toml          # wry 0.45 + tao 0.30 (the [[poc007]] harness lineage)
  src/main.rs         # arg parse, GDK_BACKEND=x11, window, event loop
  dot.yaml            # cargo-guarded build -> ~/.local/bin/preview-wv
nushell/actions/preview       # + `window [N] [--close]` verb; stop closes windows
nushell/actions/preview-test  # new tests (fake-daemon pattern, 9 existing)
nvim/plugins/preview.lua      # :PreviewStart spawns slot window
i3/config                     # for_window [class="preview-wv"] floating enable
sway/config                   # same rule (XWayland — GDK forced x11)
```

**Conventions.** Pidfile discipline mirrors the daemon's `start`
(`~/.cache/preview/wv-<N>.pid`; probe → stale-clean → spawn). Loud
errors name the fix ("rotz install preview-webview"). The wrapper is the
only entry point ([[adr0001]] / [[adr0003]] engine-not-interface). The
rotz recipe mirrors `preview/dot.yaml` (toolchain-guarded build into
`~/.local/bin`). Wrapper tests extend `nushell/actions/preview-test`
(akm-graph-test / d2-router-test pattern, fake binaries in a temp PATH).

**Anti-patterns (ship-blockers).**

- Never SIGTERM a pidfile pid without verifying `/proc/<pid>/cmdline`
  contains `preview-wv` — pid recycling kills innocent processes.
- No window spawn when the daemon is down — probe `/status` first, loud
  error pointing at `preview start`.
- No `unwrap`/panic on user input in Rust — bad args → usage message +
  nonzero exit.
- No consumer (nvim, scripts) calls `preview-wv` directly — wrapper only.

**Known limitations.** No `--display`/`--xauth` targeting flags (first
slice inherits the environment; [[poc007]] proved the mechanism when
needed). No browser fallback — loud error instead. Termux/proot ARM
untested (webkit2gtk + cargo availability unverified there). GDK backend
forced to X11 — under sway the window runs via XWayland, which is what
makes `class=` rule matching uniform across i3 and sway.

### Task 1: `preview-wv` Rust crate + rotz build

#### bd
dotfiles-hoe.1

#### type
task

#### effort
5h

#### depends
none

#### files_touched
`preview-webview/Cargo.toml`, `preview-webview/src/main.rs`,
`preview-webview/dot.yaml`

#### success_criteria
- `preview-wv <N> [--port P]` opens a window loading
  `http://127.0.0.1:<port>/preview<N>`; port resolution: flag >
  `$PREVIEW_PORT` > 4200 (maps to [[ft005]] api_surface `GET
  /preview<N>` — the client consumes it verbatim).
- Window carries WM_CLASS `preview-wv` and title `preview <N>`
  (xprop-verified) — the contract the WM float rules (Task 3) match on.
- Process exits 0 when the window is closed; bad args → usage message +
  nonzero exit.
- `rotz install preview-webview` builds `~/.local/bin/preview-wv` via a
  cargo-guarded recipe (mirrors the Go daemon's `preview/dot.yaml`
  toolchain guard).

#### edge_cases
`N` non-numeric or negative → usage error, nonzero exit. Daemon down →
window shows the connection error but the process stays up (reconnect is
the daemon-side websocket's concern). No `DISPLAY` → clean GTK init
error + nonzero exit, no panic. `--port` non-numeric → usage error.

#### test_plan
`cargo test`: arg-parse table test (N/port/defaults/invalid). Manual:
launch against a live daemon → window renders `/preview<N>`; xprop
confirms WM_CLASS; close window → exit 0; launch with `DISPLAY=` unset →
nonzero exit, no panic backtrace.

### Task 2: wrapper `window` verb + stop integration

#### bd
dotfiles-hoe.2

#### type
task

#### effort
6h

#### depends
1

#### files_touched
`nushell/actions/preview`, `nushell/actions/preview-test`

#### success_criteria
- `preview window [N]` (default 1) spawns `preview-wv` detached and
  writes `~/.cache/preview/wv-<N>.pid`; alive pid whose
  `/proc/<pid>/cmdline` matches `preview-wv` → no-op (idempotent);
  stale/mismatched pid → cleaned + respawned.
- Missing binary → error naming "rotz install preview-webview"; daemon
  not answering `/status` → error naming `preview start` (loud-error
  stance from sp013 solution).
- `preview window [N] --close` SIGTERMs the cmdline-verified pid and
  removes the pidfile; `preview stop` closes all `wv-*.pid` windows
  before stopping the daemon.

#### edge_cases
Recycled pid (alive but cmdline mismatch) → treated stale, never
killed. Double `--close` → idempotent clean message. Pidfile dir
missing → created. Concurrent `window <N>` invocations → at most one
survivor, no orphan pidfile. `--close` on never-opened N → clean
message, exit 0.

#### test_plan
Extend `nushell/actions/preview-test` with a fake `preview-wv` (sleep
stub on temp PATH, existing fake-daemon pattern): spawn writes pidfile;
second spawn no-ops; stale pidfile cleaned + respawned; cmdline-mismatch
pid NOT killed (assert stub pid untouched); `--close` terminates stub +
removes pidfile; `stop` closes all windows; missing binary error names
the rotz recipe; daemon-down error names `preview start`.
