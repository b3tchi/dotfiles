# Spec [[cat001]] [[board]]

## problem

Live d2 preview today is one `d2 --watch` process per nvim buffer
(nvim/plugins/lang-d2.lua): every file grabs a random port and its own
browser tab. N diagrams = N ports, N tabs, no overview, no way to force
a re-render or stop watchers beyond per-buffer keymaps.

Wanted: one fixed-port umbrella over all d2 previews with flat
`/{project}/{filename}` routing — project names resolved from the
project registry ([[ft001]], `projects.yaml`), `*.d2` files globbed
under each registered project path. Plus a control plane over the
watchers: list running children (file, port, clients, last compile,
last error), force reload, hard restart, stop — reachable from CLI and
nvim.

Empirical constraints (probed against d2 v0.7.1):

- `--watch` serves exactly one file per process; `--host`/`--port` pin
  the listener, but any URL path is interpreted as a board name inside
  the diagram — no native multi-file routing exists.
- Served HTML references `/static/*` root-absolute and watch.js
  connects `ws://{host}/watch` root-absolute — a path-prefix proxy
  must rewrite both, coupling the wrapper to d2's watch.js shape.

Capability boundary: a router daemon owning spawn/reap/proxy of
`d2 --watch` children plus a small control API (status, force reload
via touch, restart, stop). Consumers: the nvim d2 integration
(migrates off ad-hoc per-buffer job code in nvim/plugins/lang-d2.lua)
and ad-hoc browser/CLI use (index page of all diagrams across
projects). No existing feature covers this — a new ft### gets minted
at spec-writing.

Constraint tension: [[adr0001]] / [[adr0002]] frame nushell-vs-bash
for scripts; a long-running websocket-proxy daemon fits neither.
Orchestration stays nushell, but the daemon engine (Go agreed during
brainstorm) sits outside the frame — likely a new ADR (compiled helper
allowed when daemon/proxy semantics required).

Surveyed: [[cat001]] (workflow-tooling) is the home category;
[[im001]] / [[im002]] carry no d2 logic — the only migration target is
the nvim plugin file above.

## solution

Reverse-proxy control daemon over native `d2 --watch` children
(brainstorm option 3) — keeps d2's instant websocket reload instead of
reimplementing compile/serve/notify. Mints [[ft002]]
(d2-preview-router): a Go daemon `d2-router-d` on one fixed port
(`$D2_ROUTER_PORT`, default 4800) that builds its routing table from
the [[ft001]] project registry, serves an index of all `*.d2` files
across registered projects, lazy-spawns one child per requested
diagram on a router-assigned port (no stdout port parsing), proxies
HTTP + websocket under flat `/{project}/{filename}` routes with two
body rewrites (root-absolute `/static/*` assets, watch.js `/watch` ws
path — exact strings confirmed by the 2026-06-04 probe against d2
v0.7.1), reaps idle children, and exposes the control API: status,
force reload (touch → recompile + broadcast), hard restart, stop.

Orchestration stays nushell per [[adr0001]] / [[adr0002]] — an
idempotent `d2-router` wrapper (start/stop/status/open/reload) is a
thin API client; the Go engine itself sits outside both ADRs'
bash-vs-nu frame, so a companion ADR ("compiled helper allowed when
daemon/ws-proxy semantics are required") gets drafted during
refinement or retro. Consumers: nvim d2 integration ([[ft002]]
replaces the per-buffer job code in nvim/plugins/lang-d2.lua;
`<leader>cp` → ensure daemon + open routed URL, `<leader>cP` → force
reload) and ad-hoc browser/CLI use. Within [[cat001]].

Trade-offs accepted: watch.js rewrite couples to d2's served assets
(rewrite-miss → log warning, serve unrewritten); N child processes
bounded by the idle reaper; flat basenames collide within a project
(first wins + index warning).

Evidence: live probes of d2 v0.7.1 confirmed single-file watch,
`--port` pinning, board-name path semantics, root-absolute `/static/*`
+ `ws://{host}/watch`, and watch.js reconnect loop — the rewrite +
upgrade-proxy approach rests on observed behavior, not assumption; Go
`httputil.ReverseProxy` websocket pass-through is stock library
behavior. No PoC needed.

## plan

File tree (new + touched):

```
d2/
  dot.yaml                 # + go build install step (linux)
  router/
    go.mod                 # module d2router; dep: gopkg.in/yaml.v3
    main.go                # env config, server wiring, shutdown
    registry.go            # projects.yaml → route table, *.d2 walk
    children.go            # child lifecycle: spawn/touch/restart/reap
    proxy.go               # prefix routes, body rewrites, ws upgrade
    api.go                 # /api/* JSON + index page
    *_test.go              # per-file unit tests, fixtures/
nushell/actions/d2-router  # nu wrapper: start|stop|status|open|reload
nvim/plugins/lang-d2.lua   # migrated consumer
```

Conventions:

- Go stdlib + `gopkg.in/yaml.v3` only (registry parse — stdlib has no
  YAML). Everything else stdlib. Trivial cross-arch builds.
- Daemon binds `127.0.0.1` only; router port `$D2_ROUTER_PORT`
  (default 4800); child ports router-assigned from
  `$D2_ROUTER_CHILD_PORT_BASE` (default 4801) by free-port probe —
  never parse child stdout for ports.
- d2 binary path injectable via `$D2_ROUTER_D2_BIN` (default `d2`) —
  this is the test seam for fake children.
- Registry path injectable via `$D2_ROUTER_REGISTRY` (default
  `~/.config/project/projects.yaml`) — test seam + [[ft001]] contract.
- All child state in one mutex-guarded map keyed `{project}/{file}`.
- Force `Accept-Encoding: identity` toward children so body rewrites
  see plain text.
- Orchestration (wrapper, nvim glue) stays nushell/lua per
  [[adr0001]] / [[adr0002]]; daemon legitimacy via new ADR (Task 1).

Anti-patterns:

- No `panic` in handlers — every error becomes 4xx/5xx + JSON message.
- No hand-rolled websocket framing — `httputil.ReverseProxy` upgrade
  pass-through only.
- No shelling out to nu/bash from the daemon.
- No silent rewrite failures — expected-substring miss logs a warning
  (once per child) and serves the body unmodified.
- No `time.Sleep` polling in tests — readiness via TCP probe with
  deadline.

Known limitations:

- Body rewrites couple to d2's served watch.js shape — probed against
  v0.7.1 (`/static/*` root-absolute, `ws://{host}/watch`). d2 upgrades
  re-probe; miss degrades to unrewritten serve + warning.
- Flat basename routing: duplicate basenames within one project →
  lexically-first path wins, index flags the collision.
- [[ft001]] `ssh:` registry entries skipped — local files only.
- Build requires go toolchain on the target (rotz install guard).

Deviations (approved at refinement):

- Feature-entry spec — no `us###` / `im###`; task success criteria
  bind to [[ft002]] `## api_surface` + this spec's `## problem`.
- Daemon reads `projects.yaml` directly (read-only) per [[ft001]]
  `## data_model` documented schema instead of shelling `project
  list`.
- [[ft002]] widened in-commit with `GET /api/resolve?path=`.

### Task 3: Child manager — spawn, touch-reload, restart, reaper

#### type
feature

#### effort
8h

#### depends
Task 2

#### files_touched
- d2/router/children.go
- d2/router/children_test.go

#### success_criteria
- `ensure({project}/{file})` spawns `$D2_ROUTER_D2_BIN --watch
  --browser 0 --host 127.0.0.1 --port <assigned>` once per key —
  concurrent callers get the same child (per-key lock); readiness =
  TCP connect probe with deadline before first use.
- Port assignment probes free ports from
  `$D2_ROUTER_CHILD_PORT_BASE`; occupied ports skipped.
- `reload` = `os.Chtimes` touch on the watched file; `restart` =
  SIGTERM → wait (2s) → SIGKILL → respawn, new pid, same key/port
  policy; `stop` removes the child from the map.
- Reaper kills children with `clients == 0` idle past
  `$D2_ROUTER_IDLE` (default 30m); active children untouched.
- Child crash: `Wait` goroutine records exit error as `lastError`,
  child removed from running set; daemon shutdown kills all children
  (no orphans, no zombies).

#### edge_cases
- d2 binary missing → spawn error stored, surfaced as 502 downstream;
  watched file deleted (child lives, restart on gone file → error);
  port range exhausted → explicit error; child never opens port →
  readiness timeout → 502; double-stop race.

#### test_plan
- Fake child via `$D2_ROUTER_D2_BIN` pointing at a test helper that
  listens on the given `--port` (TestHelperProcess pattern): concurrent
  ensure → exactly one process; restart → pid changes; crash helper →
  `lastError` set; reaper with 0-client child + tiny idle → killed;
  port probe skips a port pre-occupied by `net.Listen` in the test.
