# Feature [[cat002]] [[product]]

## providing
A reusable quickshell mode-hint bar for i3/sway binding modes: a
name-pill (highlighted background + red underline + bold display name)
followed by a keyboard-hint strip (red key, cream label), rendered from
a static per-mode registry whenever the WM is in a non-default mode.
The mode set is defined by [[ft003]] (i3-config layering): `resize`,
`screenshot` (which exists solely to paint this strip for [[ft006]]
region capture — `qs-screenshot.sh` documents the dependency), and the
`$mode_system` power menu. The system provides the strip once; each bar
host supplies the current mode string and its font size. Consumed today
by the main bar (`quickshell/config/Bar.qml`); the reuse axes are
(a) per-mode hint data added in one place and (b) future bar hosts —
a sway bar, the phone `BarChin`, a second-monitor bar. Replaces the
inline duplicated mode-strip code (dead indicator block, hardcoded
colors, data/render tangle) that lived in `Bar.qml`.

## api_surface
QML components under `quickshell/config/Common/` (same import root as
`Session.qml` / [[ft008]]'s `Combo`), instantiated by each bar host:

    import "./Common"

    ModeBar {
        mode:     root.currentMode  // i3/sway mode name;
                                    // "default" renders nothing
        fontSize: root.fontSize     // host's session-aware pixel size
    }

ModeBar is an Item, NOT a Window — the host owns window chrome,
anchoring, and the mode-subscription `Process` that feeds `mode`.
`implicitWidth` tracks content; `visible` is false while `mode` is
`"default"`. ModeBar contains no Process and no Window.

ModeBarTheme is a singleton of the shared constants and the data:

    ModeBarTheme.highlight  // "#CB4B16" — key text + pill underline
                            // (highlight kept red)
    ModeBarTheme.pillBg     // "#152024"
    ModeBarTheme.fg         // "#FDF6E3"
    ModeBarTheme.muted      // "#707880"
    ModeBarTheme.font       // "Iosevka Nerd Font"

    ModeBarTheme.hints            // { resize: [{key,label}...],
                                  //   screenshot: [...], system: [...] }
    ModeBarTheme.resolve(mode)    // -> "resize"|"screenshot"|"system"
                                  //    |null (unknown)
    ModeBarTheme.hintsFor(mode)   // -> [{key,label}] rows; unknown mode
                                  //    -> [{key: "", label: mode}]
    ModeBarTheme.displayName(mode)// -> pill label; non-resize/screenshot
                                  //    modes display as "system"

Lookup is by function, not by raw map key: the system mode's IPC name is
the full `$mode_system` string (`(l)ock, (e)xit, switch_(u)ser, ...`),
matched by the `(l)ock` sniff lifted verbatim from the pre-refactor
`Bar.qml modeHints()`. Rows render with `Text.NativeRendering` to match
the bar.

## data_model
Stateless. The hints registry is static constant data owned by the
singleton; the current-mode state is owned by each host's own
mode-subscription `Process` ([[ft003]]'s i3 IPC `subscribe ["mode"]`).
ModeBar holds no state beyond its two input properties.

## sample
The main bar consuming the strip (illustrative):

    // Bar.qml — subscription stays host-side
    Process {
        command: [root.wmMsg, "-t", "subscribe", "-m", '["mode"]']
        stdout: SplitParser {
            onRead: data => root.currentMode = JSON.parse(data).change
        }
    }
    ModeBar {
        mode: root.currentMode
        fontSize: root.fontSize
        anchors { left: parent.left; top: parent.top
                  bottom: parent.bottom; leftMargin: 8 }
    }

See the pre-refactor inline implementation this is extracted from in
[quickshell/config/Bar.qml](../../quickshell/config/Bar.qml) (mode
tracking, `modeHints()`, and the "Mode hints overlay" Row).
