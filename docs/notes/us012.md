# Story [[cat001]] [[product]]

## role
[[pn001|developer]]

## want
the clip script family (`clip-store.sh`, `clip-feed.sh`, `clip-set.sh`,
`qs-clip.sh`) re-evaluated against [[adr0002]] and, where the boundary
says nushell, rewritten in nushell — as a deliberate supersession-or-
reaffirmation of the bash carve-out, not a quiet swap

## because
The [[sp016]] pivot grew the clip glue to four cooperating `#!/bin/sh`
scripts. They currently ride [[adr0002]]'s carve-outs, but the family now
composes a workflow-tooling layer, which [[adr0002]] itself says defaults
to nushell. The user explicitly deferred this to post-v1 (epic
dotfiles-egm notes, 2026-07-22): ship the store on the proven sh shape
first, revisit the language once it has survived daily use. If the
rewrite happens it is an [[adr0002]]-boundary decision worth an ADR
(supersede or reaffirm), plus a refactor story with the existing suites
as the safety net.

## acceptance_criteria
- an explicit written call per script: which [[adr0002]] condition keeps
  it bash, or that it moves to nushell — recorded as a new ADR if the
  boundary shifts
- any rewritten script passes its existing adversarial suite unchanged
  (the suites are the contract: gate order, exit codes 0/1/2/78, store
  atomicity, flock idempotency)
- no behavioral regression on the hot path: capture latency and picker
  list latency within the same bounds as the sh versions
- startup-cost condition ([[adr0002]] #4) measured, not assumed, for the
  per-event and per-keybind paths
