# Feature [[cat002]] [[product]]

## providing
A reusable quickshell keyboard-driven combobox for the i3 dialog
family: an input bar with type-to-filter, a fuzzy-scored match with
match-character highlight, an arrow-navigable row list, and the
Enter-confirm / Esc-cancel contract. Consumed by every graphical i3
dialog via the [[ft003]] keybind + quickshell host layer — the launcher
(mod+d), window switcher (mod+w), projects (mod+p), and clip-history
picker (mod+v). The system provides the control once; each dialog supplies
its own model, row renderer, and confirm action. Replaces the two divergent
copies that exist today (the launcher/switcher/projects dialog duplicated
across `quickshell/overlay/shell.qml` and `quickshell/config/Overlay.qml`,
and the separate substring-only [[ft007]] clip picker in
`quickshell/config/ClipHistory.qml`).

## api_surface
QML components under `quickshell/config/Common/` (same import root as
`Session.qml`), instantiated by each dialog:

    import "./Common"

    Combo {
        model:        <list of row objects>         // dialog-supplied
        delegate:     <row renderer Component>       // dialog-supplied
        onConfirm:    (row) => { ... }               // dialog-supplied action
        onCancel:     () => hide()
    }

Combo owns: the input `TextField`, the filtered `ListView`, arrow/Enter/Esc
key handling, and the fuzzy pipeline. Fuzzy is a standalone module reused
directly:

    Fuzzy.match(text, pattern)      -> { matched: bool, score: int, indices: [int] }
    Fuzzy.highlight(text, indices)  -> rich-text string with matched chars marked

DialogTheme is a singleton of the shared constants:

    DialogTheme.width       // 480
    DialogTheme.inputHeight // 32
    DialogTheme.inputBg     // "#152024"
    DialogTheme.rowHeight   // 32
    DialogTheme.maxRows     // 8
    DialogTheme.bodyBg      // "#222D31"
    DialogTheme.fg          // "#FDF6E3"
    DialogTheme.font        // "Iosevka Nerd Font"
    // rows use Text.NativeRendering

The filter never reorders the caller model destructively: the confirm
handler receives the row object the caller supplied, so an id-stable
consumer (the clip picker) publishes its own `row` id, never a positional
index — the [[adr0010]] publish-not-paste contract is preserved.

## data_model
Stateless. Combo holds only transient view state (current
filter string, selected row index) for the lifetime of an open dialog.
It owns no persistent store — each consumer supplies and owns its model
(PATH bins, WM tree, project registry, or the [[ft007]] clip file store).

## sample
launcher consuming the control (illustrative):

    Combo {
        model: pathBins                              // string list
        delegate: BinRow {}
        onConfirm: (row) => execProc.exec(row.name)
        onCancel: () => root.hide()
    }

See the current pre-refactor implementations the control is extracted from:
the fuzzy pipeline and dialog chrome in
[quickshell/config/Overlay.qml](../../quickshell/config/Overlay.qml), and
the id-stable selection contract in
[quickshell/config/ClipHistory.qml](../../quickshell/config/ClipHistory.qml).
