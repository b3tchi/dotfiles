# Implementation [[cat001]] [[product]]

## solves
[[us002|register a location as a project and navigate to it easily]]

## approach
Compose the existing nushell project module ([[ft001]]) as the single
primitive the developer interacts with. Registration is `project add
<name> [path]` writing to `~/.config/project/projects.yaml`; navigation
is `project go <name>` (cd + tmux session switch/create); discovery is
`project list` from the shell and the `tmux-project` fzf picker bound
to tmux `prefix+p`. No story-specific glue beyond binding the fzf
picker — `[[ft001]]` already carries the registry, persistence, and
tmux session-group handling. Trade-off: identifier is a short name,
not a fuzzy match across paths; the developer must remember (or pick
from list) rather than typing partial paths.

## features
- [[ft001|project registry and navigation module]]

## data_model
None — owned by [[ft001]] (`~/.config/project/projects.yaml`).

## api_surface
None added — consumes [[ft001]]'s nushell command surface
(`project add`, `project list`, `project go`, `project remove`,
`project config`) plus the tmux `prefix+p` keybinding.

## components
- `nushell/actions/tmux-project` — fzf project picker bound to tmux
  `prefix+p`
- `tmux/tmux.conf` — keybinding for `prefix+p` → `tmux-project`
