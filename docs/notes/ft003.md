# Feature [[cat002]] [[product]]

## providing
One shared i3 base configuration (keybinds, workspaces, modes,
appearance) consumed by every platform that runs i3 — native linux,
proot, and wsl (xrdp session from [[poc002]]) — plus thin per-platform
overlays for autostarts and display-specific settings. Eliminates
per-platform config drift; a change to the base lands everywhere on
relink.

## api_surface
i3 `include` directive (i3 >= 4.20). The base config ends with
`include ~/.i3/config.d/*.conf`; rotz links exactly one platform
overlay into `~/.i3/config.d/` via handlebars conditionals in
[i3/dot.yaml](../../i3/dot.yaml). Contract: base owns keybinds /
workspaces / modes / colors and never calls `exec`; overlays own all
`exec` / `exec_always` autostarts (tray, compositor, shell overlays,
wallpaper, autolock) and display-specific tweaks. WSL overlay starts
clean: no quickshell, no picom, no tray apps.

## data_model
Stateless — plain i3 config files. Layout:
[i3/config](../../i3/config) (base),
`i3/config.d/native.conf` (linux + proot overlay, planned),
`i3/config.d/wsl.conf` (xrdp overlay, planned).

## sample
```
# i3/config (base) — last line
include ~/.i3/config.d/*.conf

# i3/dot.yaml links
links:
  config: ~/.i3/config
  {{#if env.WSL_DISTRO_NAME}}
  config.d/wsl.conf: ~/.i3/config.d/wsl.conf
  {{else}}
  config.d/native.conf: ~/.i3/config.d/native.conf
  {{/if}}
```

## components
- [i3/config](../../i3/config)
- [i3/dot.yaml](../../i3/dot.yaml)
- [xrdp/xinitrc](../../xrdp/xinitrc) (wsl session entry consuming the shared config)
- `i3/config.d/` overlays (planned)
