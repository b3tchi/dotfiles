# Feature [[cat002]] [[product]]

## providing
One shared i3 base configuration (keybinds, workspaces, modes,
appearance) consumed by every platform that runs i3 — native linux,
proot, and wsl (xrdp session from [[poc002]]) — plus thin per-platform
overlays for autostarts and display-specific settings. Eliminates
per-platform config drift; a change to the base lands everywhere on
relink.

## api_surface
i3 `include` directive (i3 >= 4.20). The base config includes
`~/.i3/config.d/*.conf` near the top; rotz links exactly one platform
overlay into `~/.i3/config.d/` via handlebars conditionals in
[i3/dot.yaml](../../i3/dot.yaml). Contract: base owns keybinds /
workspaces / modes / colors plus the quickshell autostart (all
platforms); overlays own platform autostarts (tray, compositor,
wallpaper, autolock, keyboard layout) and display-specific tweaks.
Caveat: variables set in included overlays are INVISIBLE to the main
file (i3 include limitation) — platform-varying values used by base
binds go through X resources: base reads
`set_from_resource $mod i3wm.mod Mod4`; the wsl session entry
([xrdp/xinitrc](../../xrdp/xinitrc)) merges `i3wm.mod: Mod1` via xrdb
before exec i3.

## data_model
Stateless — plain i3 config files. Layout:
[i3/config](../../i3/config) (base),
[i3/config.d/native.conf](../../i3/config.d/native.conf) (linux + proot),
[i3/config.d/wsl.conf](../../i3/config.d/wsl.conf) (xrdp session).

## sample
```
# i3/config (base) — near top
set_from_resource $mod i3wm.mod Mod4
include ~/.i3/config.d/*.conf

# i3/dot.yaml links
links:
  config: ~/.i3/config
  {{#if env.WSL_DISTRO_NAME}}
  config.d/wsl.conf: ~/.i3/config.d/wsl.conf
  {{else}}
  config.d/native.conf: ~/.i3/config.d/native.conf
  {{/if}}
```
