# Story [[file-preview]] [[product]]

## role
[[pn001|developer]]

## want
open a node's / element's source file at my cursor in nvim from the
embedded akm graph or an embedded d2 diagram inside the preview window —
triggered by a dedicated open-in-editor affordance (a small shortcut
glyph button on the node's label), not by bare node selection

## because
[[ft005]] shipped the receiving half of this reverse channel — the shell
listens for a `postMessage` and issues `POST /open` — but against an
*assumed* contract: [[ft002]] and [[ft004]] do not emit the selection
event yet, so the click-to-open does nothing end to end. Closing this
makes the bidirectional preview actually bidirectional.

## acceptance_criteria
- [[ft004]] renders a small open-in-editor glyph button on the node
  label (visible on hover/selection); activating the glyph — not bare
  node click, which keeps its existing isolate behavior — emits the
  `postMessage` carrying the backing file path; [[ft002]] gains an
  equivalent per-element affordance for d2
  (routed through `idea-extend` on each feature — sp009 / sp010)
- the emitted message contract (origin, shape, `window.top` vs
  `window.parent` target) matches [[ft005]]'s shell listener, or the
  listener is reconciled to match — verified against `preview/static/app.js`
- selecting a node/element opens the correct source file in the running
  nvim; a selection with no backing file is a silent no-op
- an emit from an unexpected origin is ignored by the shell listener
