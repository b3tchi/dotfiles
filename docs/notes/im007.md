# Implementation [[cat002]] [[product]]

## solves
none — feature-promotion spec [[sp018|mode-bar-reusable-component]]; the
mode-strip smells were undocumented `Bar.qml` glue captured directly at
spec stage and promoted to [[ft009]], with no source `us###`.

## approach
One extraction pass moves the i3/sway mode-hint strip (name-pill +
keyboard-hint row shown during `resize` / `screenshot` / system mode)
out of `quickshell/config/Bar.qml` and onto the new capability [[ft009]]
minted for this purpose: a `ModeBar` item plus a `ModeBarTheme`
singleton under `quickshell/config/Common/` — the same
DialogTheme + Combo split [[im006]] shipped for [[ft008]] (commit
89ea553), scaled down to one consumer. The host keeps ownership of the
mode state (the i3 IPC `subscribe ["mode"]` `Process` stays in
`Bar.qml`) and of window chrome/anchoring; [[ft009]] owns the render
(pill + hint strip) and the constants + per-mode hint registry.

Three smells die in the migration: the dead "Mode indicator" Rectangle
(nested under a `visible: currentMode === "default"` Row while itself
`visible: currentMode !== "default"` — unreachable) is deleted; the
lowercase hardcoded colors (`#cb4b16`/`#152024`/`#fdf6e3`/`#707880`)
move into `ModeBarTheme` as named constants (highlight kept red
`#CB4B16` for key text + pill underline); and the inline `modeHints()`
map + display-name ternary move into the singleton as data +
`resolve()/hintsFor()/displayName()` lookups, lifted verbatim including
the `(l)ock` sniff for the long `$mode_system` IPC name and the
`[{key: "", label: mode}]` unknown-mode fallback. Parity is the
contract: extracted colors and geometry reproduce the pre-refactor
render exactly ([[sp017]] precedent). `ModeBarTheme` stays separate from
[[ft008]]'s `DialogTheme` — bar/dialog theme independence chosen over
deduplicating overlapping palette values.

## features
- [[ft009|mode-bar-control]] — the strip itself, minted at [[sp018]]
  refinement; `proposed` until the spec ships ([[ft008]] precedent)
- [[ft003|i3-config-layering]] — mode set (`resize`, `screenshot`,
  `$mode_system`) and the i3 IPC subscription consumed unchanged; no
  config or bind edits
- [[ft006|region-capture]] — untouched; its `screenshot` mode keeps
  painting through the extracted strip

## data_model
none new — the hints registry is static constant data in
`ModeBarTheme`; current-mode state stays in `Bar.qml`'s existing
subscription `Process`

## api_surface
no new external surface. i3 modes, binds, and IPC are byte-compatible
before and after. Internal QML surface added is [[ft009]]'s
`ModeBar` / `ModeBarTheme` under `quickshell/config/Common/`.
