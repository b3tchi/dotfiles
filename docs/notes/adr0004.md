# ADR [[cat002]] [[product]]

## title
xrdp + xorgxrdp + llvmpipe as the WSL2 full-desktop remoting stack

## context
WSL2 needed a full i3 desktop reachable over RDP; WSLg only offers
per-window RAIL. [[poc001]] invalidated upstream weston's RDP backend
(in-frame cursor lag, encode pipeline seconds behind, GL-renderer
segfault under WebGL load, black GL windows). [[poc002]] validated
xrdp + xorgxrdp: client-side cursor, session survives disconnect,
snappy llvmpipe desktop with WSLg-parity performance on real
workloads (PrusaSlicer usable). Wayland-side alternatives
(gnome-remote-desktop, Plasma krdp) bring foreign desktop stacks and
no i3; wlroots compositors have no RDP backend at all.

## decision
Adopt xrdp + xorgxrdp serving a full i3 session with llvmpipe
software rendering as the WSL2 desktop-remoting stack. RFX codec
preferred over H.264 (4:2:0 chroma subsampling washes flat UI
colors; H.264 kept as fallback). Port 3391. Sessions are exclusive
with the WSLg sway session, flipped via wsl-wm-switch. Packaged as
[[ft003]]-consuming xrdp rotz package.

## consequences
Positive: native i3 over the network, disconnect-resilient sessions,
full 4:4:4 color, good interactive feel without GPU. Negative:
CPU-only rendering caps heavy 3D/WebGL (~17-25 FPS aquarium);
xrdp/xorgxrdp live in AUR (manual rebuild on xorg-server bumps);
xorgxrdp requires picom --no-use-damage or windows render black;
auth is the Linux password with TLS deferred — trusted-LAN exposure
only until hardened.
