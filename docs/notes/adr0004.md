# ADR [[cat002]] [[product]]

## title
xrdp + xorgxrdp + llvmpipe as the WSL2 full-desktop remoting stack

## context
A full i3 desktop must be reachable over RDP on WSL2; WSLg offers only
per-window RAIL. Wayland-side RDP servers (weston-rdp,
gnome-remote-desktop, krdp) cannot host i3, and weston-rdp was
invalidated by [[poc001]]. xrdp + xorgxrdp was validated by [[poc002]].

## decision
Full-desktop remoting on WSL2 goes through xrdp + xorgxrdp serving a
native i3 X11 session, rendered on llvmpipe.

## consequences
Locks the WSL2 remote desktop onto the X11 path while wayland remoting
matures elsewhere. Heavy 3D stays CPU-bound. xrdp/xorgxrdp come from
AUR, so xorg-server bumps need rebuilds. Implementation surface lives
in the xrdp package and [[ft003]]; evidence in [[poc002]].
