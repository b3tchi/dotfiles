# Implementation [[cat001]] [[product]]

## solves
[[us006|visualize-note-relations]]

## approach
Compose [[ft004|akm-graph]] — built fresh by this story as its first
consumer (now `accepted`) — per the [[adr0003]] compiled-daemon gate,
with the user surface staying nushell per [[adr0001]]. The daemon owns
walking, parsing, serving, and live-pushing the graph; story-specific
glue is only the `akm graph` passthrough subcommand in the akm CLI and
the rotz registration. Single-file cosmos vendoring is proven by
[[poc003]] (pin 1.6.1). Trade-off: viewer is browser-based rather than
TUI — accepted, force-directed GPU layout has no TUI equivalent.

Shipped shape (as built): viewer assets served via `//go:embed static`
(cwd-independent, offline). The graph is swapped atomically under an
`sync.RWMutex` so concurrent `/api/graph` reads never see a half-built
graph. Live reload is fsnotify per-directory (non-recursive: new
subdirs added on CREATE) with a ~300 ms debounce coalescing editor
save-bursts into one rebuild + one WS broadcast; a `Hub` drops
slow/stuck WS clients (non-blocking send) so one reader can't stall the
loop. The `/watch` WS surface uses `gorilla/websocket` (build-time dep,
baked into the binary — offline-safe) rather than a hand-rolled RFC6455
framer; the `akm-graph` wrapper mirrors the [[ft002]] daemon
conventions (port-probe → pidfile → detached launch, stale-pidfile
cleanup). Follow-up filed during the e2e walk: symlinked hub note
`docs/notes/akm.md` is skipped by the symlink guard (bd `dotfiles-2ry`).
