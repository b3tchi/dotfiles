# Implementation [[cat001]] [[product]]

## solves
[[us006|visualize-note-relations]]

## approach
Compose [[ft004|akm-graph]] — built fresh by this story as its first
consumer (feature is `proposed`) — per the [[adr0003]] compiled-daemon
gate, with the user surface staying nushell per [[adr0001]]. The
daemon owns walking, parsing, serving, and live-pushing the graph;
story-specific glue is only the `akm graph` passthrough subcommand in
the akm CLI and the rotz registration. Single-file cosmos vendoring
is proven by [[poc003]] (pin 1.6.1). Trade-off: viewer is
browser-based rather than TUI — accepted, force-directed GPU layout
has no TUI equivalent.

## features
- [[ft004|akm-graph]]

## data_model
none — read-only over the notes tree; all runtime state lives in
[[ft004]] (in-process graph, rebuilt on fsnotify change).

## api_surface
none beyond [[ft004]] — the `akm graph` subcommand in the akm CLI is
a thin passthrough to the ft004 nu wrapper.
