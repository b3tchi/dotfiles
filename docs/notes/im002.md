# Implementation [[cat001]] [[product]]

## solves
[[us001|mark project as remote so terminals open against the remote host]]

## approach
Compose three already-shipped primitives plus one additive widening of
[[ft001]]. The project module branches on a new optional `ssh: <profile>`
field in `projects.yaml`: absent → today's local flow; present →
delegate to `tmux-to-workstation connect` with `--cwd <path>` carrying
the remote working directory. ssh user/host/port resolution stays in
`~/.ssh/config.d/<profile>` — the registry only stores the profile
alias.

The remote working directory is honored at the tmux layer: a new
optional `cwd` positional on `nushell/actions/tmux-start` threads
through to `tmux new-session -c <dir>`, so the remote session's
default-directory is the project path. `tmux-to-workstation` gains a
`--cwd` flag that propagates the value into the remote `tmux-start`
invocation over ssh.

Trade-offs:

- ssh field is a plain string (profile name), not a record. Keeps the
  yaml schema readable and pushes connection params to ssh config — the
  proper single source of truth. Cost: a remote project requires an
  out-of-band `~/.ssh/config.d/<profile>` setup; `project add` does not
  bootstrap it.
- Dispatch branches on presence of `ssh`, not on a separate boolean.
  One field per concept. Adding a boolean would let the two fields
  disagree.
- Auto-cd skips remote entries. Auto-ssh on every shell open is
  surprising and breaks offline; remote is opt-in via explicit
  `project go`.
- `--ssh` requires explicit `path` at `project add` time. PWD-default
  for remote would silently register the developer's local PWD as the
  remote project dir — wrong shape.

**WM-keybinding extension ([[sp002]]).** The same `ssh`-branch dispatch
is reused from a second trigger: the WM `mod+Return` keybinding.
[[ft001]] gains a public `lookup <name>` reader so a WM-side dispatcher
can read the registry without re-parsing yaml. A new nu action
`nushell/actions/wm-launch-terminal` resolves the focused workspace via
`wm-current-workspace`, looks up the entry via `project lookup`, and
branches:

- **No `ssh` (or workspace unregistered)** → today's local trigger:
  `TMUX_TARGET=$(tmux-start start 0 <ws>) <term-cmd>`.
- **`ssh` set** → `TMUX_TARGET=$(tmux-to-workstation connect --no-attach
  -g <ws> --cwd <path> -- <ssh>) <term-cmd>`. The new `--no-attach`
  flag on `tmux-to-workstation connect` prints the local thin session
  id (e.g. `thin_5`) and exits without attaching.

`distro/bashrc` (carve-out #2 under [[adr0002]]) gains a one-line
pattern guard inside the existing TMUX_TARGET attach block: values
matching `^thin_` attach via `tmux -L thin`; all other non-empty values
keep the default-socket attach (no regression on local workspaces).
Both branches share the single `TMUX_TARGET` env-protocol; the socket
selection lives entirely in bashrc's prefix match.

`sway/config.d/default:119` and `i3/config:38` simplify from the
embedded `tmux-start` inline to a one-liner `exec
~/.local/bin/wm-launch-terminal $term`.

## features
- [[ft001|project registry and navigation module]] — widened additively
  with the optional `ssh` field, `--ssh` flag on `add`, `ssh` column on
  `list`, dispatch branch in `go`, and skip-remote in the auto-cd
  `export-env` block. Status stays `stable`; api_surface gains
  backwards-compatible additions only.

## data_model
None new — extends [[ft001]]'s owned state
(`~/.config/project/projects.yaml`) with one optional field per entry:

```yaml
projects:
  projectx:
    path: /home/user/repos/myproj    # remote when ssh set, local otherwise
    ssh: workstation                  # optional; ~/.ssh/config.d/workstation
```

Absent `ssh` → local entry (today's shape). Present `ssh` → remote
entry, `path` interpreted on the remote side. Backwards-compatible:
existing local entries keep working untouched.

## api_surface
None added at the module level — composes:

- [[ft001]] widened: `project add <name> <path> --ssh <profile>`,
  dispatch in `project go`, `ssh` column in `project list`,
  skip-remote in `export-env`, plus public `project lookup <name>`
  reader (sp002) returning `{ path }` / `{ path, ssh }` / `null`.
- `tmux-to-workstation connect -g <name> --cwd <path> -- <ssh-alias>`
  (new `--cwd` flag from sp001) plus `--no-attach` flag (sp002) that
  prints the local thin session id to stdout and exits without
  attaching.
- `tmux-start start <mode> <pinned> <session> <uid> <cwd>` (new
  optional 5th positional from sp001).
- `wm-launch-terminal <term_cmd: string>` (sp002) — WM-side dispatcher
  invoked by sway/i3 bindsym. Resolves focused workspace, looks up via
  `project lookup`, sets `TMUX_TARGET` to either the local
  `tmux-start` session or the remote thin session id, then execs the
  terminal.
- TMUX_TARGET env-protocol: `distro/bashrc` (sp002) inspects the value
  and attaches via `tmux -L thin` when the prefix is `^thin_`, default
  socket otherwise. Single env contract for both dispatch branches.

ssh alias resolution is delegated to ssh itself (no parsing in the
module).

## components
- `nushell/scripts/project/mod.nu` — schema round-trip, `add` flag,
  `list` column, `go` dispatch, `export-env` skip, plus public
  `lookup <name>` reader (sp002)
- `nushell/scripts/project/README.md` — schema + UX doc update
- `nushell/actions/tmux-start` — optional `cwd` positional → `-c <dir>`
  on every `new-session` invocation
- `nushell/actions/tmux-to-workstation` — `--cwd <path>` flag,
  propagated to remote `tmux-start` over ssh; plus `--no-attach`
  (sp002) that prints the thin session id and exits before attach
- `nushell/actions/wm-launch-terminal` (sp002) — WM-side dispatcher
  composing `wm-current-workspace` + `project lookup` + either
  `tmux-start` (local) or `tmux-to-workstation connect --no-attach`
  (remote). Sets `TMUX_TARGET` and execs the terminal command.
- `nushell/dot.yaml` (sp002) — wires the new action symlink (windows
  + linux link entries) and chmod +x in linux installs.cmd
- `distro/bashrc` (sp002) — `^thin_` pattern guard in the existing
  TMUX_TARGET attach block: `tmux -L thin attach -t $TMUX_TARGET`
  when prefix matches, default socket otherwise
- `sway/config.d/default:119` (sp002) — bindsym `$mod+Return exec
  ~/.local/bin/wm-launch-terminal $term`
- `i3/config:38` (sp002) — bindsym `$mod+Return exec
  ~/.local/bin/wm-launch-terminal st`
- `~/.ssh/config.d/<profile>` — out-of-band (user-managed) source of
  truth for connection parameters; named by the `ssh` field but not
  written by this module

## roundtrip

End-to-end flow for `project go projectx` where projectx is registered
as `{ path: /home/user/repos/myproj, ssh: workstation }`.

```text
┌──────────┐  ┌────────────┐  ┌──────────────────────┐  ┌─────┐  ┌────────────────┐  ┌─────────────┐
│ dev      │  │ project.go │  │ tmux-to-workstation  │  │ ssh │  │ remote         │  │ remote tmux │
│ (shell)  │  │ (ft001)    │  │   connect            │  │     │  │ tmux-start     │  │ (server)    │
└────┬─────┘  └─────┬──────┘  └──────────┬───────────┘  └──┬──┘  └───────┬────────┘  └──────┬──────┘
     │ project go   │                    │                  │             │                  │
     │ projectx     │                    │                  │             │                  │
     ├─────────────►│                    │                  │             │                  │
     │              │ read yaml          │                  │             │                  │
     │              │ → ssh field set    │                  │             │                  │
     │              │ → branch=REMOTE    │                  │             │                  │
     │              │ tmux-to-workstation connect           │             │                  │
     │              │   -g projectx                         │             │                  │
     │              │   --cwd /home/user/repos/myproj       │             │                  │
     │              │   -- workstation                      │             │                  │
     │              ├───────────────────►│                  │             │                  │
     │              │                    │ ssh workstation  │             │                  │
     │              │                    │ "tmux-start start 0 projectx '' /home/user/repos/myproj"
     │              │                    ├─────────────────►├────────────►│                  │
     │              │                    │                  │             │ new-session -d   │
     │              │                    │                  │             │   -t projectx    │
     │              │                    │                  │             │   -s projectx_0  │
     │              │                    │                  │             │   -c /home/user/repos/myproj
     │              │                    │                  │             ├─────────────────►│
     │              │                    │                  │             │◄─target──────────┤
     │              │                    │◄─"projectx_0:0"──┤◄────────────┤                  │
     │              │                    │ create LOCAL thin session:                        │
     │              │                    │   tmux -L thin new-session -d -s thin_N           │
     │              │                    │   cmd = "ssh -t workstation TMUX_TARGET=projectx_0 bash -li"
     │              │                    │   set-option @remote_target=projectx_0:@<wid>     │
     │              │                    │ tmux -L thin attach -t thin_N                     │
     │              │◄───detach + reattach via thin socket──┤             │                  │
     │              │                    │ (thin pane runs the ssh command)                  │
     │              │                    │                  │ ssh -t      │                  │
     │              │                    │                  │ TMUX_TARGET=projectx_0 bash -li│
     │              │                    │                  ├────────────►│                  │
     │              │                    │                  │             │ bash -l reads    │
     │              │                    │                  │             │ TMUX_TARGET,     │
     │              │                    │                  │             │ attach -t        │
     │              │                    │                  │             │ projectx_0       │
     │              │                    │                  │             ├─attach──────────►│
     │ ◄────────────dev sees remote tmux pane, cwd = /home/user/repos/myproj─────────────────┤
```

Layer roles:

- `project go` — yaml branch; remote → dispatch to tmux-to-workstation.
- `tmux-to-workstation connect` — bootstraps the remote tmux session
  via ssh, then wraps the ssh pane in a local thin tmux window (via
  `tmux -L thin`) carrying connection metadata.
- Remote `tmux-start` — creates the remote session at `--cwd` path
  using tmux `new-session -c <dir>`.
- Remote tmux server — owns the actual session state; the local thin
  pane is just a viewport.

Detach / reconnect path (already shipped in tmux-to-workstation):

- ssh drops → thin pane goes `dead` or `broken` → `tmux-to-workstation
  revive` respawns the ssh command in-place and reattaches to the same
  `projectx_0` session on the remote tmux server. Remote state survives
  the disconnect.

### Second trigger: WM `$mod+Return` (sp002)

Same dispatch primitive (`tmux-to-workstation connect`) reached from a
different entry point — the WM keybinding rather than `project go`.

```text
┌──────────┐  ┌────────────────────┐  ┌─────────────┐  ┌──────────────┐  ┌────────────┐  ┌─────────────┐
│ dev      │  │ wm-launch-terminal │  │ project     │  │ tmux-to-     │  │ $term      │  │ bashrc      │
│ ($mod+   │  │ (sp002)            │  │ lookup      │  │ workstation  │  │ bash login │  │ TMUX_TARGET │
│  Return) │  │                    │  │ ([[ft001]]) │  │ --no-attach  │  │            │  │ guard       │
└────┬─────┘  └─────────┬──────────┘  └──────┬──────┘  └──────┬───────┘  └──────┬─────┘  └──────┬──────┘
     │ bindsym fires    │                    │                │                  │                │
     ├─────────────────►│ ws=wm-current-     │                │                  │                │
     │                  │    workspace       │                │                  │                │
     │                  │ lookup ws ─────────►│                │                  │                │
     │                  │◄── { path, ssh } ──┤                │                  │                │
     │                  │ ssh field set      │                │                  │                │
     │                  │ → REMOTE branch    │                │                  │                │
     │                  │ connect --no-attach \              │                  │                │
     │                  │   -g ws --cwd path -- ssh           │                  │                │
     │                  ├──────────────────────────────────►│                  │                │
     │                  │                    │                │ bootstrap remote │                │
     │                  │                    │                │ tmux + create    │                │
     │                  │                    │                │ local thin_N     │                │
     │                  │                    │                │ session, NO      │                │
     │                  │                    │                │ attach           │                │
     │                  │◄────── "thin_N" (stdout) ─────────┤                  │                │
     │                  │ TMUX_TARGET=thin_N $term           │                  │                │
     │                  ├────────────────────────────────────────────────────►│                │
     │                  │                                    │                  │ bash login,    │
     │                  │                                    │                  │ TMUX_TARGET    │
     │                  │                                    │                  │ inherited     │
     │                  │                                    │                  ├──────────────►│
     │                  │                                    │                  │                │ =~ ^thin_
     │                  │                                    │                  │                │ → tmux -L
     │                  │                                    │                  │                │   thin attach
     │                  │                                    │                  │                │   -t thin_N
     │ ◄─────────────── dev sees thin pane attached to thin_N, ssh already opened to remote ────┤
```

Local branch (workspace not registered, or registered without `ssh`):
identical structure, but `wm-launch-terminal` calls
`tmux-start start 0 <ws>` instead of `tmux-to-workstation connect`, and
`TMUX_TARGET` carries a default-server session id (e.g. `myws_0`).
bashrc's `^thin_` guard does not match → default-socket attach. No
regression on local workspaces.

Single env-protocol shape across both branches: `wm-launch-terminal`
sets `TMUX_TARGET`; bashrc owns the socket selection via the `^thin_`
prefix. The dispatch primitive is unchanged from sp001 — sp002 only
added the WM trigger and the bashrc multiplex.
