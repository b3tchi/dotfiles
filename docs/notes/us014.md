# Story [[cat001]] [[product]]

## role
[[pn001|developer]]

## want
`clip-store/` reachable from the meta-packages, so a fresh
`rotz install meta-linux` / `meta-wsl-i3` brings in `clipnotify` and the
clipboard history works without a manual `rotz install clip-store`

## because
No `meta-*` dot.yaml depends on `clip-store/`, so a fresh install links
the i3 configs (whose autostarts invoke `clip-store.sh`) but never
installs `clipnotify` — the loop dies at its own `command -v` check
(exit 69) on every machine that didn't run the sp016 pipeline by hand.
Noted by dotfiles-egm.5 as a **pre-existing gap class**: app dots added
outside a meta refresh silently miss fresh installs, and clip-store is
the newest instance. Worth fixing as the class (audit which app dots the
metas actually pull) rather than the instance alone.

## acceptance_criteria
- `meta-linux` and `meta-wsl-i3` (the i3-running environments) depend on
  `clip-store`; environments without a WM (meta-termux) do not
- a fresh-install simulation (or clean container/VM run) of an i3 meta
  ends with `clipnotify` installed and the store loop able to start
- an audit list of other app dots referenced by linked configs but
  absent from every meta's `depends`, each either added or explicitly
  waived
- the gap class gets a durable note (bd remember or CLAUDE.md) so new
  dots are checked against the metas at creation time
