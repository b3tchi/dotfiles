# Story [[project-config]] [[remote-work]] [[product]]

## role
[[pn001|developer]]

## want
mark a project as remote so opening that project automatically lands me
in its remote home directory — no manual connection step

## because
i don't want to manually handle the connection every time i work on a
remote-hosted project — the project knows where it lives, so opening it
should just put me there

## acceptance_criteria
- project carries a flag identifying it as remote-hosted and where its
  home directory lives
- opening a remote-flagged project lands automatically in that remote
  home directory, no manual connection step
- projects not flagged keep opening locally with no behavior change
- the remote location is configured once per project and persists
  across sessions
