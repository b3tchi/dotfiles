# Implementation [[cat002]] [[product]]

## solves
[[us015|picker-overlay-visual-parity]]

## approach
One refactor pass moves all four i3/quickshell dialogs — launcher
(mod+d), window switcher (mod+w), projects (mod+p), clip picker
(mod+Shift+v) — onto the shared combobox control [[ft008]] minted for
this purpose: a `Combo` item plus `Fuzzy` and `DialogTheme` singletons
under `quickshell/config/Common/`. Each dialog keeps ownership of its
model source, row renderer, confirm action, and special modes; [[ft008]]
owns the interaction contract (type-to-filter, arrow nav, Enter/Esc) and
the parity chrome constants. The clip picker gains fuzzy filter and
match-char highlight as a side effect, satisfying the [[us015]] parity
AC; the drifted `overlay/shell.qml` duplicate of `config/Overlay.qml`
collapses into a thin `ShellRoot` wrapper (in-repo relative symlinks
expose `Overlay.qml` + `Common/` to the overlay profile), folding in the
`find -L` symlink fix as the two copies merge. Hosting is unchanged:
ClipHistory stays in the MAIN quickshell instance on every session type,
and the native-vs-RDP overlay instance split ([[adr0004]]) is preserved.
The picker still publishes and never pastes ([[adr0010]]): Combo's
confirm handler passes the caller's row object back, so the clip
consumer publishes its own opaque `row` id, never a positional index.

Two execution choices refined the shipped shape. The clip picker and
the switcher run `filterMode: "external"` (not Combo's built-in fuzzy
sort): the built-in sort alphabetizes on an empty query, which would
destroy the clip picker's newest-first order and the switcher's MRU
order, so both keep a caller-side `Fuzzy.match` filter that preserves
their own ordering while still gaining the subsequence match + escaped
highlight. And the selected row's primary text is bold across all four
delegates (post-ship polish) — the active entry already carried a
background + accent marker; `font.bold: isSelected` makes it read at a
glance without touching geometry.

## features
- [[ft008|shared-combobox-dialog-control]] — the control itself; its
  api_surface was widened at [[sp017]] refinement with optional
  per-dialog knobs (textOf, filterMode, inputVisible, maxVisibleRows,
  comboWidth, altConfirm, imperative next/prev/confirmCurrent) so the
  switcher and clip picker fit without forking the contract
- [[ft007|shared-clipboard-history]] — clip picker backend consumed
  unchanged: `qs-clip.sh list/set` surface, 0/1/2/78 exit contract,
  opaque ids end to end
- [[ft003|i3-config-layering]] — keybind + host delivery layer consumed
  unchanged; no bind or autostart edits

## data_model
none new — Combo holds only transient view state (filter string,
selected index) per open dialog; every model stays consumer-owned
(PATH bins, WM tree, project registry, [[ft007]] clip file store)

## api_surface
no new external surface. IPC targets (`launcher`, `switcher`,
`projects`, `cliphistory`) and `qs-overlay.sh` / `qs-clip.sh` verbs are
byte-compatible before and after. Internal QML surface added is
[[ft008]]'s `Combo` / `Fuzzy` / `DialogTheme` under
`quickshell/config/Common/`.
