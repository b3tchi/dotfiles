# Implementation [[cat001]] [[product]]

## solves
[[us003|create-diagrams]]

## approach
Author d2 diagrams as plain text in neovim and preview them live
through the [[ft002]] d2-preview-router daemon. In a registered
[[ft001]] project, `<leader>cp` ensures the daemon is up, resolves the
buffer path to its routed URL via the control API, and opens the
browser — d2's native websocket live-reload keeps the preview in sync
on every save; `<leader>cP` forces a re-render. Outside registered
projects the plugin falls back to the legacy per-buffer `d2 --watch`
job (kept as named functions). Per [[adr0003]] the daemon engine is
compiled Go; orchestration stays nushell per [[adr0001]]. Trade-off:
preview UX couples to the router being installed (`rotz install d2`),
with the fallback covering unregistered or router-less hosts.

## features
- [[ft002|d2-preview-router]]

## data_model
none — stateless consumer; routes resolved at runtime from [[ft002]]
(`/api/resolve`), no state owned by this card.

## api_surface
none added beyond [[ft002]]. User-facing surface is two nvim keymaps:
`<leader>cp` (open live preview) and `<leader>cP` (force reload) on
`FileType d2`.

  filetype plugin: router flow + legacy per-buffer watch fallback
