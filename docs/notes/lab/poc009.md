# PoC [[cat002]] [[board]]

## hypothesis
QtWebEngine (Qt6 Chromium webview) loads the akm-graph viewer (localhost:4810) and renders the force-graph correctly, and — being the same Qt6 runtime as quickshell — is the stack-aligned host, worth its RAM if it integrates with the existing qs shell.

## method
Scratch /tmp/poc-qtweb (throwaway). Required installing qt6-webengine 6.11.1 (94MB download / 282MB installed — bundled Chromium; most deps already present via quickshell qt6-base/ffmpeg). Harness: 15-line QML (Window + WebEngineView url=localhost:4810), run via qml6 (no compile). Ran on display :0 (AMD radeonsi X11, QTWEBENGINE_DISABLE_SANDBOX=1) against live akm-graph-d (45 nodes/157 links). Measured RSS across qml6 + QtWebEngineProcess; screenshotted :0.

## result
VALIDATED render — cleanest of the three tested. Full force-graph renders correctly (nodes, links, HTML label pills, legend chips, status), chrome-less window (no url bar). RSS ~625MB total across 4 procs (qml6 277MB + 3 QtWebEngineProcess: 59+59+230MB — Chromium multi-process). Heaviest of the three hosts. No errors.

## recommendation
Three-way host comparison complete: Wry/WebKitGTK ~400MB (poc008) < Servo ~430MB (poc007) < QtWebEngine ~625MB. QtWebEngine is HEAVIEST (~1.5x WebKitGTK, Chromium multi-proc) AND adds a 282MB Chromium dep. Its ONE real advantage: same Qt6 runtime as quickshell — the viewer could embed as a native quickshell surface (floating window / panel) sharing the qs session model, rather than a standalone app. So the choice is: quickshell integration + cleanest render (QtWebEngine, ~625MB) vs lightest standalone (Wry, ~400MB, 1.4MB binary, engine preinstalled) vs no adoption at all. CONFIRMED across all three: no webview cuts RAM vs a browser — engine is the cost, ranging ~400-625MB. Since force-graph already fixed the GPU heat (dotfiles-olm), webview host = optional. If quickshell-integrated viewer/preview surface is the goal, QtWebEngine despite RAM; if a lean standalone window, Wry. For the sp008 preview bridge, prefer Qt Quick NATIVE renderers (image/video/pdf/3D) and reserve QtWebEngine only for actual HTML — never pay ~625MB for a thumbnail.
