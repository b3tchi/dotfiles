# PoC [[cat002]] [[board]]

## hypothesis
cosmic-comp can run headless (Wayland) on WSL Arch and expose its session over
COSMIC's *native* RDP to a Windows client, rendering an interactive COSMIC
desktop.

## method
Research-only spike, no code / no worktree (the contradiction is settled by
primary sources, so no install was warranted). Probed local env + fetched
primary sources 2026-06-25.

- Local: Arch on WSL2, kernel 6.18 microsoft-standard-WSL2, WSLg active
  (Weston, WAYLAND_DISPLAY=wayland-0, DISPLAY=:0). xrdp 0.10.6 + xorgxrdp
  0.10.5 + freerdp 3.27 already installed; cosmic NOT installed.
- pop-os/cosmic-comp issue #584 "[Feature Request] Implement RDP Server".
- olafkfreund/cosmic-ext-rdp-server README (v0.1.0, 2026-02-09) — the only
  COSMIC-over-RDP stack that exists.

## result
INVALIDATED on two independent points.

1. Native RDP — not upstream. Issue #584 open since 2024-06-28, no PR, no
   merge. cosmic-comp ships no RDP server. The only COSMIC-over-RDP path is a
   third-party stack: cosmic-ext-rdp-server + cosmic-comp-rdp fork +
   xdg-desktop-portal-cosmic. Not native, not in extra.

2. Headless — unsupported. cosmic-comp backends are KMS / X11 / Winit only; no
   headless backend (Smithay has offscreen, cosmic-comp does not expose it).
   The third-party rdp-server is explicitly "not headless-compatible" — it
   captures a *live* display via ScreenCast portal + PipeWire. WSL exposes a
   DRM render node only (no KMS modesetting / connectors), so even the KMS
   backend brings up no display. WSL also lacks VAAPI/NVENC (x264 software
   fallback exists, but the live-display requirement is the blocker, not the
   encoder).

The literal triple — Wayland + native RDP + headless — is impossible on WSL as
of 2026-06.

## recommendation
Do not pursue cosmic native-RDP headless on WSL now. Keep i3 + xrdp over X11
(adr0004 / adr0005) as the WSL remote-desktop path.

Watch signal: pop-os/cosmic-comp#584 (native RDP server). Re-spike only when
that merges AND a headless / virtual-output backend lands — both are required,
either alone is insufficient.

Achievable-but-rejected alternatives (recorded so they aren't re-litigated):
- Nested cosmic-comp in WSLg (Winit/X11, software render, reaches Windows via
  WSLg's own RDP) — gives full DE in a window, but not headless and not native
  RDP; nested-compositor segfaults reported on WSLg (microsoft/wslg#1069).
- Third-party COSMIC RDP stack — real COSMIC-over-RDP, x264 software, but heavy
  (3 repos + compositor fork), needs a live display (not headless), not native,
  zero WSL testing.
