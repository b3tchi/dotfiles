# PoC [[cat002]] [[board]]

## informs
[[sp008|cursor-driven-preview-bridge]]

## hypothesis
Running N instances of the same webview engine (Servo / Wry-WebKitGTK / QtWebEngine) scales its hardware (RAM) requirement sub-linearly — shared engine libraries mean the marginal cost of an extra window is far below the single-instance RSS — which matters for sp008 multi-preview-window viability.

## method
Scratch /tmp/poc-3way (throwaway; servoshell re-fetched, wry recompiled, qt.qml). Launched 1/2/3 concurrent instances of each engine on display :0, all pointed at akm-graph-d :4810 (47 nodes/160 links). Two metrics per step: summed RSS (ps grep per engine) AND system-used delta (MemTotal-MemAvailable from /proc/meminfo). System total 17908 MiB. CAVEAT: measured ~8-11s after each launch under 3-way CPU/software-EGL contention = MID-LOAD footprints, understated vs a fully-settled single instance (a fully-rendered servo is ~400-430MB per poc007, vs ~253MiB caught mid-load here).

## result
Summed RSS scales ~linearly per instance: Servo 254/505/758 MiB (1/2/3); Wry+WebKit 393/782/1169 MiB (3/6/9 procs); QtWebEngine 608/1229/1835 MiB (4/8/12 procs). BUT actual system-used delta for the full 3x is far smaller: Servo ~0-100 MiB, Wry +339 MiB, QtWebEngine +596 MiB. The gap = shared engine .so pages mapped once (copy-on-write), which RSS double-counts across every process. True marginal cost of an extra same-engine window ~= its private JS heap + canvas buffers (~110-200 MiB), NOT the full single-instance RSS. Per-instance private breakdown stayed constant (e.g. Wry each = poc-wry 131 + WebKitNetwork ~50 + WebKitWeb ~207 MiB; Qt each = qml6 ~270 + 3 QtWebEngineProcess 59+59+224).

## recommendation
Multi-window of the SAME engine is viable and cheaper than N x single-instance: shared code pages mean the incremental window costs its heap+canvas (~110-200MB), not a fresh ~400-600MB engine. Ranking unchanged (Servo lightest summed, Qt heaviest, ~2.4x Servo at 3x). For sp008: N concurrent preview windows on ONE chosen engine scale fine on this 18GB box (3x QtWebEngine = +596MB real). Do NOT mix engines (that pays each engine's base separately). Combined with poc007-010: the tiered design still wins — native Qt renderers (poc010, ~155MB shared runtime) for image/video/code/3D avoid the web engine entirely; reserve ONE web engine (QtWebEngine for quickshell-integration, or Wry standalone) for HTML, and multiple HTML preview windows then share that engine cheaply. Re-measure fully-settled absolutes if precise per-window numbers are needed; the sub-linear SHARING shape is the durable result.
