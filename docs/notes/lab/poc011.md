# PoC [[cat002]] [[board]]

## informs
[[sp016|clipcat-backend-swap]]

## hypothesis
clipse (savedra1/clipse) can replace clipcat as the sp016 clipboard backend:
memory-only history on $XDG_RUNTIME_DIR, gopass leak caught, id-keyed get,
byte-exact multiline round-trip (beats dotfiles-i9i), deterministic
newest-first list (beats dotfiles-8il).

## method
Scratch dir /tmp/poc-clipse (discarded). go install clipse v1.2.1. Listener
on throwaway Xvfb :98, config pointed historyFile at /tmp tmpfs, real xclip
copies, JSON history + -output-all raw/unescaped inspected byte-level via od.
Kill-logic read from module cache source (shell/common.go, constants.go).

## result
MIXED CAPABILITIES, ONE FATAL DEFECT.
PASS: ordering deterministic newest-first with ns timestamps (beats 8il).
PASS: -output-all raw emits JSON-quoted strings -> reversible escaping;
unescaped emits true raw bytes (beats i9i).
FAIL: no id-keyed get (only whole-history dump; JSON file readable directly).
FAIL: memory-only broken — image entries written to $XDG_CONFIG_HOME/clipse/
tmp_files/ regardless of historyFile; plain TEXT copies misclassified as
phantom PNG entries, so the persistent leak fires even for text.
UNTESTED (moot): gopass filter — excludedApps is app-name based, cannot see
xclip-mediated gopass ownership.
FATAL (discovered live): shell.KillExisting() runs on every `clipse -listen`
start; isClipseExe() does strings.Contains("wl-paste", exeName) — BACKWARDS
containment — so it SIGKILLs every process whose short name is a substring
of "wl-paste": st, wl, paste, a, t... Killed the user's st terminal (and
the Claude session inside it) three times during the poc, masquerading as
xrdp session drops. meta-wsl-i3 uses st as its terminal — disqualifying.

## recommendation
Reject clipse as sp016 backend. The st-killer bug alone disqualifies it on
any st-based setup; the persistent image leak and text-as-image
misclassification independently break the memory-only guarantee. Do not
revisit before upstream fixes isClipseExe and the tmp_files leak. For 8il/
i9i remediation prefer probing clipcat gRPC raw reads or a bespoke
clipnotify+file-store, both informed by sp016.
