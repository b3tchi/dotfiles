# PoC [[cat001]] [[board]]

## informs
[[sp006|akm-graph-cosmos-viz]]

## hypothesis
@cosmograph/cosmos can be vendored as a single standalone browser JS
file (go:embed-able, no CDN, no runtime imports) that renders an
interactive force graph from JSON nodes/links offline.

## method
Scratch dir `/tmp/poc-cosmos-vendor` (discarded). `npm install
@cosmograph/cosmos` → stable 1.6.1 (npm latest dist-tag is
2.0.0-beta.26 — beta line avoided, pin 1.6.1). `bun build entry.js
--format esm --minify` with entry `export { Graph } from
"@cosmograph/cosmos"` → one 830 KB ESM file, zero `import`/`require`
of anything external. Test page: `new Graph(canvas, {...})` +
`setData(10 nodes, 9 links)`, served via `python3 -m http.server`,
loaded in headless Windows Chrome (`--headless=new --disable-gpu
--dump-dom`, swiftshader WebGL).

## result
VALIDATED. Page title reported `OK nodes=10` —
`graph.getNodePositions()` returned positions for all 10 nodes after
simulation start. Fully offline: only localhost fetches of the
vendored bundle. Bundle is one self-contained file, 830 KB minified
(ESM), suitable for go:embed.

## recommendation
Adopt single-file vendoring for [[sp006]] / [[ft004]]: pin
@cosmograph/cosmos 1.6.1, bundle once with bun (or esbuild) at vendor
time, commit the bundle under the daemon static dir, serve via
go:embed. Do not depend on the 2.0.0-beta line until it goes stable;
v1 `Graph(canvas, config)` + `setData(nodes, links)` API is
sufficient for nodes/links/color/size/hover.
