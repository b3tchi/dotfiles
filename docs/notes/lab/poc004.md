# PoC [[cat002]] [[board]]

## hypothesis
gnome-remote-desktop in headless mode on WSL Arch (no /dev/dri, software render)
spawns a Wayland GNOME session served over RDP that a client renders
interactively — native-Wayland + headless + connect-from-cold RDP, the combo
cosmic ([[poc005]]) and KRdp could not deliver.

## method
Live build on this WSL Arch (kernel 6.18 microsoft-standard-WSL2, systemd on,
WSLg). Packages: gnome-shell 50.2, gnome-remote-desktop 50.1, mutter 50.2,
gnome-control-center 50.3, nautilus, pipewire/wireplumber. Client: FreeRDP
3.27 (sdl-freerdp3) local + Windows mstsc. Reference recipe:
github.com/techneut92/wsl-gnome-rdp-installer (Fedora/Debian; mechanism
adapted to Arch + grd --headless self-managed mode). Chain:
1. `modprobe vgem` → /dev/dri/renderD128 (vgem.ko prebuilt for this kernel).
   EGL then reports Device #0 EGL_EXT_device_drm + render_node — satisfies
   grd's non-null DRM-render-node requirement.
2. `gnome-shell --headless --virtual-monitor 1280x800 --wayland
   --wayland-display wayland-grd --no-x11` → mutter builds gbm renderer on
   vgem (software/llvmpipe), virtual monitor Meta-0.
3. pipewire + wireplumber up; mutter exposes org.gnome.Mutter.ScreenCast +
   RemoteDesktop on the session bus.
4. grd config: openssl TLS cert/key → `grdctl --headless rdp set-tls-cert/key`,
   `set-credentials jan <pass>`, `set-port 3390`, `set-auth-methods
   credentials`, `enable`.
5. `GALLIUM_DRIVER=llvmpipe LIBGL_ALWAYS_SOFTWARE=1
   /usr/lib/gnome-remote-desktop-daemon --headless` → "RDP server started" 3390.
6. Connect: `mstsc /v:localhost:3390` (WSL2 localhost-fwd) or sdl-freerdp3.

## result
VALIDATED end-to-end.
- mutter headless init OK on vgem node, software GL (llvmpipe).
- grd daemon: first crashed at EGL — tried Zink-on-dzn ("Zink requires
  nullDescriptor feature"); fixed by forcing software GL env. Then listens 3390.
- FreeRDP: NLA/NTLM auth passed, gdi_init_ex BGRA32, rdpgfx channel; server
  logged "RDPGFX CapsAdvertise: Accepting ... H264 AVC444" + CLIPRDR caps.
- Windows mstsc: connected; GNOME desktop renders, input works (right-click
  menu), apps launch (Settings/control-center) after installing
  gnome-control-center and with XDG_CURRENT_DESKTOP=GNOME.

Gotchas (each cost real time):
1. vgem not loaded by default + vanishes on WSL restart → needs
   /etc/modules-load.d/vgem.conf. No vgem → /dev/dri absent → grd EGL fails.
2. grd daemon MUST force software GL (GALLIUM_DRIVER=llvmpipe,
   LIBGL_ALWAYS_SOFTWARE=1) else Zink-on-dzn EGL crash. Bake into unit env.
3. RDP cred = grd's own (grdctl set-credentials), NOT PAM/Linux password.
4. Linux freerdp client defaults to Kerberos (realm ATHENA.MIT.EDU) →
   KRB5_CONFIG=/dev/null to force NTLM. mstsc (Windows SSPI) unaffected.
5. Apps need XDG_CURRENT_DESKTOP=GNOME (gnome-control-center refuses otherwise;
   gnome-shell sets it for apps it launches).
6. Bare gnome-shell --headless = no panel/dock; Super grabbed by mstsc → use
   top-left Activities hot-corner or add a dock extension.
7. mutter "Failed to add device before DEVICE_READY" — grd virtual-input
   injection hiccup; input worked but worth watching.
8. Encode is software RFX/llvmpipe only (no /dev/dri hwaccel) → CPU-bound.
9. Port 3390 (not Windows-RDP 3389); reachable via mstsc localhost:3390.

## recommendation
ADOPT gnome-remote-desktop headless as the native-Wayland remote-desktop option
for WSL — only stack delivering native-Wayland + headless + connect-from-cold
RDP (cosmic [[poc005]] and KRdp both failed this). Productionize:
- /etc/modules-load.d/vgem.conf (persist render node).
- systemd user units (headless gnome-shell + grd, software-GL env) +
  `loginctl enable-linger` for boot autostart.
- rotz dot.yaml folded into the WSL meta-packages.
- Add a dock (Super-free launching); investigate DEVICE_READY input warning.
Coexists with i3+xrdp (adr0004/adr0005) on a separate port. Tradeoff: GNOME =
heavier, Wayland-native, future-proof (no X11 EOL); i3+xrdp = lighter, tiling,
X11 (EOL pressure, Plasma 6.8 Oct 2026).
