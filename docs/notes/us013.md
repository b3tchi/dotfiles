# Story [[cat001]] [[product]]

## role
[[pn001|developer]]

## want
the `copyq/` and `clipcat/` trees removed from the repo once the
[[sp016]] file-store backend has survived one cycle of daily use

## because
[[sp016]] kept both superseded backends in-tree as rollback layers —
links removed, nothing autostarted — exactly one cycle deep: `copyq/`
(layer 1, the [[sp014]] backend) and `clipcat/` (layer 2, validated then
falsified in execution). Dead trees that can be re-linked are a rollback
asset for days and a confusion source forever; `clipcat/test-clipcat.sh`
has already served its purpose as the adaptation source for
`test-clip-store.sh`. The decision chain they witness is preserved in
[[adr0011]], so the trees carry no unique knowledge.

## acceptance_criteria
- at least one cycle of real daily use on the file store across native
  `:0`/`:10` and WSL with no rollback needed
- `copyq/` and `clipcat/` directories deleted, including their dot.yaml,
  configs, and test suites; no rotz dot references them
- the commented rollback blocks in `i3/config.d/native.conf` (and any
  other config prose pointing at re-enabling them) removed in the same
  change
- repo-wide grep shows remaining copyq/clipcat mentions only in
  immutable history: archived specs, pocs, ADRs, closed bd notes
