# Feature [[cat001]] [[cat002]] [[product]]

## providing
A localhost Go daemon (`preview-d`) plus its mandatory nushell wrapper
that renders any file on demand and drives a live floating webview which
follows the editor cursor. Two route modes compose: a stateless
`/file/<path>` that renders one file as-is (renderer chosen by the path's
type) and a stateful `/preview<N>` live window that hot-swaps which file
it shows as the cursor moves. Bidirectional — the editor pushes the
cursor's file in, and selecting a node/element in the webview pushes a
path back out to open in nvim. Renders code, markdown, images, video, and
STL, and embeds [[ft004]] (akm graph) as a cross-origin iframe and
[[ft002]] (d2 diagrams) through a same-origin frame-header-stripping
proxy ([[adr0006]]). Consumed by nvim and nnn.

Note: the node/element → nvim reverse channel ships on the shell side
(a `postMessage` listener → `POST /open`) against an *assumed* message
contract — [[ft002]] / [[ft004]] do not yet emit the selection event.
That page-side emit and the final contract are pending sp009 / sp010.

## api_surface
HTTP on `localhost:$PREVIEW_PORT` (default 4200):

- `GET /file/<path>` — render one file as-is; renderer selected by the
  path's extension/type: code (syntax HTML), markdown (HTML), image
  (thumbnail, full-res on `?full`), video (poster frame, `<video>` on
  `?live`), STL (orbit viewer, raw bytes on `?full`). An akm-zettel path
  resolves to a cross-origin `<iframe>` of [[ft004]]; a `.d2` path
  resolves to a `<iframe>` of the same-origin `/d2embed/` proxy of
  [[ft002]] ([[adr0006]]). Embedded backends are lazy-spawned if not
  already listening.
- `GET /d2embed/<path>` — same-origin reverse-proxy of the [[ft002]]
  d2 page for `<path>`, with frame-blocking headers stripped ([[adr0006]]);
  path-jailed like `/file/<path>`.
- `GET /preview<N>` — live webview window `N`: a shell that loads the
  current `/file/<path>` and hot-swaps it. The same `/preview<N>` path
  upgrades to a websocket for the per-window redraw push. Multiple `N` =
  multiple concurrent windows, each an isolated slot (path + client set).
- `POST /preview<N>` body `{path}` — set window `N`'s current path (the
  editor cursor signal); the daemon pushes it to that window.
- `POST /open` body `{path}` — open the path in the running editor via
  `nvim --server $NVIM --remote` (the reverse channel); path-jailed +
  arg-vector exec, `424` when `$NVIM` is unset.
- `GET /status`, `POST /stop` — health snapshot / graceful stop
  (ft002/ft004 parity).

Mandatory nushell wrapper ([[adr0003]]) at
[nushell/actions/preview](../../nushell/actions/preview), the user-facing
API: `preview start | stop | status | open <path> | send <path> [--window N]`.
No consumer invokes the binary directly.

## data_model
Ephemeral, no persistence. In-memory session map: window slot `N` →
`{ current path, set of websocket clients }`, swapped atomically under a
mutex (the [[ft004]] hub pattern). Video poster frames are cached
per-path under a mutex (ffmpeg-extracted, first-frame); the cache is
currently unbounded — eviction is deferred (bd `dotfiles-e6o`). Embedded
peer daemons are lazy-spawned once under a double-checked lock and
health-polled with a timeout. Config via env `$PREVIEW_PORT` (default
4200). All viewer libs (three.js / `<model-viewer>`, syntax assets) are
compiled into the binary via `go:embed` — offline, no CDN, no
target-side build ([[ft004]] precedent).

## sample
```nu
preview start                       # spawn daemon + open window 1 (--app webview)
preview send ~/notes/us006.md       # push a file to window 1 -> it redraws
preview open ~/notes/us006.md       # (reverse) open that path back in nvim
```

nvim side (`CursorHold` autocmd calls the wrapper):

```lua
vim.api.nvim_create_autocmd("CursorHold", {
  callback = function() vim.system({ "preview", "send", vim.fn.expand("%:p") }) end,
})
```
