# Feature [[cat001]] [[cat002]] [[product]]

## providing
A shared clipboard history for the graphical i3 sessions: every *text* copy
made on any live X session lands in one in-memory history, and a quickshell
window browses it and republishes the picked entry to the clipboard of every
live display. Consumed by the i3 keybind layer via [[ft003]] — the picker
keybind lives in the shared base config, the backend autostarts in the
per-platform overlays.

Solves the problem that each X display owns an independent CLIPBOARD
([[cat002]]), so a copy on native `:0` is invisible on the xrdp `:10` session
([[adr0004]]) and vice versa, and neither keeps history.

SCOPE: text only; images and rich clipboard are not captured. The picker
**copies, it does not paste** — see [[adr0010]]. Distinct from [[ft006]]
(region-capture), which only drops a screenshot's file *path* on the
clipboard; no overlap.

History is memory-only by construction — nothing reaches disk, and the
history dies with the server at logout.

## api_surface
Keybind for the human, plus a two-verb shell contract for programmatic
callers (`quickshell/qs-clip.sh`):

    $mod+Shift+v          toggle the picker on the active session
      type                filter the list
      Enter               publish the selected row, close
      Esc                 close, publish nothing

    qs-clip.sh toggle     open/close the picker on the derived session
    qs-clip.sh list       history as "<row>\t<preview>" lines, newest first
    qs-clip.sh set <row>  publish row <row> to CLIPBOARD+PRIMARY

Exit contract of `set` (propagated verbatim from `clip-set.sh`; consumers
must branch on all three, not on zero/nonzero):

    0   on CLIPBOARD+PRIMARY of every live display, each observed serving
        it back
    1   precondition failed, NO selection written anywhere — safe to retry
    2   partial write, clipboard state indeterminate

Session targeting is **derived, never inherited**: the target is resolved
from the live quickshell instances (via `/proc/<pid>/environ` plus a
`cliphistory` IPC probe), because both `:0` and `:10` are permanently live.
An inherited `DISPLAY` is honoured only when it matches a real instance;
`QS_CLIP_DISPLAY` forces one explicitly.

## data_model
No persistent state — that is the point.

    copyq history        in the running server's memory only, per-tab
                         `store_items=false`; nothing on disk, cleared at
                         logout
    row indices          positional, newest-first; they SHIFT as new items
                         arrive, so a row is only valid for the instant it
                         was read

The backend is addressed as a plain `copyq` with no environment juggling: the
server socket lives inside `~/.config/copyq`, so relocating the config dir
would move the socket and break every client.

## sample
Copy something on the xrdp session, then pick it up on the native desktop:

    # on :10 — any ordinary copy
    $mod+Shift+v          # on :0, picker opens listing the :10 copy at row 0
    <type to filter>
    Enter                 # entry is now on CLIPBOARD+PRIMARY of :0 and :10

Programmatic use of the same path:

    qs-clip.sh list                      # 0<TAB>ssh-keygen -t ed25519 ...
    qs-clip.sh set 0; echo $?            # 0

## components
- `copyq/dot.yaml` — rotz install + links; its header is the **client
  contract** and documents the memory-only and secret-filter mechanisms.
- `copyq/copyq.conf` — per-tab `store_items=false` (memory-only).
- `copyq/commands.ini` — the automatic command that drops items advertising
  `application/x-kde-passwordManagerHint`. NOTE: inert against password
  managers that publish no hint atom — see `dotfiles-cyg` and [[sp015]].
- `i3/scripts/clip-feed.sh` — native-desktop-only `:10 → :0` feeder (the sole
  dual-display host). Carries its **own** TARGETS secret gate plus a
  post-read re-check, because `copyq add` bypasses copyq's automatic commands
  entirely; `flock` single-instance, timeout-guarded, explicit `DISPLAY=`.
- `i3/scripts/clip-set.sh` — publishes a history row to CLIPBOARD+PRIMARY on
  every live display; owns the 0/1/2 exit contract.
- `quickshell/qs-clip.sh` — the whole non-UI layer (session derivation,
  history parsing, capping, preview truncation) in sh so it is testable
  without an X server.
- `quickshell/config/ClipHistory.qml` — presentation only; never talks to
  copyq or `clip-set.sh` directly, only through `qs-clip.sh`.
- `i3/config` — `$mod+Shift+v` bind (base, all platforms per [[ft003]]).
- `i3/config.d/native.conf` — copyq server + feeder autostart.
- `i3/config.d/wsl.conf` — copyq server only; single-display, no feeder.

Delivered by [[sp014]]. Test suites: `copyq/test-copyq.sh`,
`i3/scripts/test-clip-feed.sh`, `i3/scripts/test-clip-set.sh`,
`quickshell/test-clip-history.sh`, `i3/scripts/test-clip-integration.sh` —
all headless (Xvfb) and mutation-tested.
