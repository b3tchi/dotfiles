# Feature [[cat001]] [[cat002]] [[product]]

## providing
A shared clipboard history for the graphical i3 sessions: every *text* copy
made on any live X session lands in a per-display file store, and a
quickshell window browses it and republishes the picked entry to the
clipboard of every live display. Consumed by the i3 keybind layer via
[[ft003]] — the picker keybind lives in the shared base config, the store
loops autostart in the per-platform overlays.

Solves the problem that each X display owns an independent CLIPBOARD
([[cat002]]), so a copy on native `:0` is invisible on the xrdp `:10`
session ([[adr0004]]) and vice versa, and neither keeps history.

The backend is a **bespoke clipnotify file store** ([[adr0011]]) — no
daemon. One `clip-store.sh` loop per display blocks poll-free on
`clipnotify -s clipboard`, gates password-manager copies on the owner's
TARGETS list (both hint-atom spellings, plus a post-read TOCTOU re-check),
and writes each capture as a sequence-numbered file. Shipped by [[sp014]]
(copyq), backend swapped by [[sp016]] (copyq → clipcat → bespoke store;
see [[adr0011]] for why vendor managers kept failing on this surface).

SCOPE: text only; images and rich clipboard are not captured. The picker
**copies, it does not paste** — see [[adr0010]]. Distinct from [[ft006]]
(region-capture), which only drops a screenshot's file *path* on the
clipboard; no overlap.

Memory-only is a **filesystem-level property**: the store lives under
`$XDG_RUNTIME_DIR` (tmpfs, 0700, destroyed by logind at last logout) and
nowhere else; every store script refuses to start (exit 78) when
`$XDG_RUNTIME_DIR` is unset rather than fall back to a persistent path.

## api_surface
Keybind for the human, plus a two-verb shell contract for programmatic
callers (`quickshell/qs-clip.sh`):

    $mod+Shift+v          toggle the picker on the active session
      type                filter the list
      Enter               publish the selected entry, close
      Esc                 close, publish nothing

    qs-clip.sh toggle     open/close the picker on the derived session
    qs-clip.sh list       history as "<id>\t<preview>" lines, newest first
                          (reverse-lexicographic filename sort —
                          deterministic; two calls over an unchanged store
                          emit byte-identical output)
    qs-clip.sh set <id>   publish entry <id> to CLIPBOARD+PRIMARY; the
                          source display is this process's own session,
                          passed down explicitly

One layer down, the publisher (`i3/scripts/clip-set.sh`):

    clip-set.sh <id> [src-display]

The source display is REQUIRED (positionally or via
`CLIP_SET_SRC_DISPLAY`) — never defaulted from inherited `$DISPLAY`. The
store is per display, so the same id routinely names unrelated entries in
`:0`'s and `:10`'s stores; guessing the source wrong would publish the
wrong content *successfully*.

Exit contract of `set` (propagated verbatim through `qs-clip.sh`;
consumers must branch on all of them, not on zero/nonzero):

    0   on CLIPBOARD+PRIMARY of every live display, each observed serving
        it back
    1   precondition failed, NO selection written anywhere — safe to retry
    2   partial write, clipboard state indeterminate
    78  configuration failure ($XDG_RUNTIME_DIR unset) — EX_CONFIG, the
        family convention every store script shares

The feeder (`i3/scripts/clip-feed.sh`, native desktop only) bridges `:10 →
:0` by writing fed entries directly into the destination display's store
dir, named by `CLIP_FEED_DST` (`CLIP_FEED_SRC` names the watched display).
A file write cannot own an X selection, so feeding structurally never
touches the destination's clipboard. `CLIP_FEED_DST_SOCKET` (the
clipcat-era daemon socket) is refused loudly, not silently ignored.

Ids are **opaque**: consumers match them for equality and existence only,
never parse, order on (beyond sorting), or do arithmetic on them
(`ClipHistory.qml` carries them as raw strings — a `parseInt` truncated
them to numbers once, dotfiles-g5b).

Session targeting of `toggle` is **derived, never inherited**: the target
is resolved from the live quickshell instances (via `/proc/<pid>/environ`
plus a `cliphistory` IPC probe), because both `:0` and `:10` are
permanently live. An inherited `DISPLAY` is honoured only when it matches a
real instance; `QS_CLIP_DISPLAY` forces one explicitly.

## data_model
The store contract (defined in `i3/scripts/clip-store.sh`'s header; every
client is written against it):

    store dir   $XDG_RUNTIME_DIR/clip-store/<display>/    0700, tmpfs
    entry       NNNNNN.clip — six-digit zero-padded monotonic seq, raw
                payload bytes exactly as the owner served them: no
                trailing-newline games, no escaping. `get` is `cat` of a
                file — byte-exact by definition.
    id          the filename; opaque to consumers
    ordering    lexicographic filename order IS capture order; newest is
                the highest seq — deterministic by construction
    work files  dotfiles (`.wip.tmp`, `.feed.wip.tmp`, `.tgt`) plus any
                in-flight `*.tmp`; consumers read only `??????.clip`
                names, so a mid-write file is never a visible entry

Entries appear atomically: payload to a `.tmp` in the same dir, `ln`(2)ed
into the final name. `ln` fails on an existing name instead of clobbering,
so two writers into one store (the display's own loop + the feeder) resolve
a seq collision as a retry, not a lost entry. Consecutive identical
captures dedup against the newest entry; `CLIP_STORE_CAP` /
`CLIP_FEED_DST_CAP` prune oldest (default 100).

No persistent state — that is the point. Plaintext exists on tmpfs for the
session lifetime and survives SIGKILL; only logind teardown clears it
(same exposure as both prior backends).

## sample
Copy something on the xrdp session, then pick it up on the native desktop:

    # on :10 — any ordinary copy
    $mod+Shift+v          # on :0, picker opens listing the :10 copy first
    <type to filter>
    Enter                 # entry is now on CLIPBOARD+PRIMARY of :0 and :10

Programmatic use of the same path:

    qs-clip.sh list                # 000007.clip<TAB>ssh-keygen -t ed25519…
    qs-clip.sh set 000007.clip; echo $?    # 0

## components
- `clip-store/dot.yaml` — rotz install of `clipnotify` (official `extra`)
  + `xclip`; its header restates the store contract for downstream
  writers. Links nothing; the loop script is linked by `i3/dot.yaml`.
- `i3/scripts/clip-store.sh` — the backend: one loop per display,
  poll-free on `clipnotify`, **defines the store contract**. TARGETS
  secret gate before any payload read (both `x-kde-passwordManagerHint`
  spellings, bare and `application/`-prefixed) + post-read TOCTOU
  re-check; fails closed. The watcher for the next event is pre-spawned
  before the current one is handled, so the loop is never deaf during a
  read. NOTE: the gate is inert against password managers that publish no
  hint atom — see `dotfiles-cyg` and [[sp015]].
- `i3/scripts/clip-feed.sh` — native-desktop-only `:10 → :0` feeder; a
  second writer into the same store, same atomic-`ln`/dedup/prune shape.
  Carries its **own** two gates (same both-spelling TARGETS pre-check +
  re-check), because a direct store write bypasses the destination loop's
  watcher-side gate entirely; `flock` single-instance, timeout-guarded,
  explicit `DISPLAY=`.
- `i3/scripts/clip-set.sh` — publishes one store entry to
  CLIPBOARD+PRIMARY on every live display; owns the 0/1/2/78 exit
  contract; requires the source display explicitly.
- `quickshell/qs-clip.sh` — the whole non-UI layer (session derivation,
  store listing, preview truncation, id shape-check) in sh so it is
  testable without an X server ([[adr0002]]).
- `quickshell/config/ClipHistory.qml` — presentation only; never touches
  the store or `clip-set.sh` directly, only through `qs-clip.sh`; carries
  ids as opaque strings.
- `i3/config` — `$mod+Shift+v` bind (base, all platforms per [[ft003]]).
- `i3/config.d/native.conf` — `clip-store.sh :0`, `clip-store.sh :10`
  ([[adr0004]]) + feeder, all `exec_always`, flock-idempotent.
- `i3/config.d/wsl.conf` — one loop (`:10`), no feeder.
- Rollback layers (one cycle, then delete): `copyq/` and `clipcat/` stay
  in-tree with links removed; neither is started by any config.

Delivered by [[sp014]], backend replaced by [[sp016]] per [[adr0011]].
Test suites: `i3/scripts/test-clip-store.sh`,
`i3/scripts/test-clip-feed.sh`, `i3/scripts/test-clip-set.sh`,
`quickshell/test-clip-history.sh`, `i3/scripts/test-clip-integration.sh` —
all headless (Xvfb), mutation-tested, negative-controlled, and fenced off
the live session's store (isolated `XDG_RUNTIME_DIR`).
