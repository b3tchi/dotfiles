# Feature [[cat001]] [[product]]

## providing
A single-port HTTP daemon (`akm-graph-d`, Go) that renders the whole
AKM zettel graph as an interactive force-directed visualization using
the GPU-accelerated cosmos library (@cosmograph/cosmos). It walks
`docs/notes/**.md` + `docs/board.md`, extracts frontmatter and
wikilink edges, serves an embedded viewer page (vendored cosmos.js via
go:embed — offline, no CDN, no build step on target), exposes the
graph as machine-readable JSON, and pushes live updates over websocket
when note files change. Consumed by the developer (browser audit of
the knowledge graph) and by scripts/skills reading `/api/graph` for
link analysis (orphans, dangling refs).

## api_surface
HTTP on `localhost:$AKM_GRAPH_PORT` (default 4810):

- `GET /` — viewer page: cosmos canvas, nodes colored by zettel type,
  sized by degree, hover tooltip with id / alias / status; dangling
  wikilink targets rendered as ghost nodes.
- `GET /api/graph` — JSON `{nodes: [{id, type, status, alias,
  degree, ghost}], links: [{source, target}]}` — deduped edges.
- `WS /watch` — pushes the rebuilt graph JSON on file change
  (fsnotify on the notes tree).
- `GET /api/status` — daemon health: pid, root, node/link counts,
  last rebuild.
- `POST /api/stop` — graceful shutdown.

Nu wrapper `akm graph` ([[adr0001]] / [[adr0003]] mandate):
`akm graph start|stop|open|status` — idempotent launcher (pidfile +
log under `~/.cache/akm-graph/`), thin API client, `open` launches
the browser at the viewer page.

## data_model
No persistent store. In-process graph rebuilt from the notes tree on
start and on every fsnotify change: node id = zettel id (filename
stem) or dangling link target (ghost), type from filename prefix
(us/sp/im/ft/adr/cat/pn/poc) or hub, status + first alias from
frontmatter, degree computed; edges = deduped `[[wikilink]]`
occurrences source→target.

## sample
```nushell
akm graph start                 # launch daemon (idempotent)
akm graph open                  # → http://localhost:4810/
akm graph status                # pid, node/link counts
http get http://localhost:4810/api/graph
  | get nodes | where ghost     # dangling-link report from a script
```

## components
Shipped ([[sp006]], Go module `akmgraph`, go 1.21):

- `akm-graph/parser.go` — `WalkNotes` (frontmatter + `[[wikilink]]`
  extraction; skips symlinks — see follow-up), `nodeTypeFromID`
- `akm-graph/graph.go` — `BuildGraph` / `BuildGraphFromRoot`: nodes
  (typed + hub), deduped edges, dangling-target ghost nodes, degree
- `akm-graph/server.go` — HTTP mux, `//go:embed static`, atomic graph
  swap under `sync.RWMutex`, `/api/graph` `/api/status` `/api/stop`,
  `/watch` WS handler + read/write pumps, `StartWatcher`
- `akm-graph/watch.go` — `Hub` (broadcast, drop-on-full slow client),
  `Watcher` (fsnotify per-dir, ~300 ms debounce, new-subdir pickup)
- `akm-graph/main.go` — `--root` / `$AKM_GRAPH_ROOT`, port 4810 /
  `$AKM_GRAPH_PORT`, graceful shutdown
- `akm-graph/static/` — `index.html`, `app.js`, single-file vendored
  `cosmos-bundle.js` (cosmos 1.6.1 via [[poc003]], go:embed)
- `nushell/actions/akm-graph` — nu wrapper (`start|stop|status|open`),
  linked to `~/.local/bin/akm-graph` by `nushell/dot.yaml`
- `akm-graph/dot.yaml` — rotz install: guarded `go build` →
  `~/.local/bin/akm-graph-d` (src via `akm-root`)

Runtime deps (build-time only, baked into the static binary —
offline-safe): `github.com/fsnotify/fsnotify` v1.9.0,
`github.com/gorilla/websocket` v1.5.3.
