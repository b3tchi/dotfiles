# Implementation [[cat001]] [[cat003]]

## solves
[[us003|rotate service credentials without downtime]]

## approach
Extend the existing ft002 vault-secrets client to support
*dual-version reads* with a short overlap. A rotation command writes a
new version to Vault, then signals consumers (via a lightweight
file-mtime poll on `/etc/acme/secrets.rev` or an in-process TTL cache,
TBD in spec) to refresh. The `secret(name)` API keeps returning the
*previous* value for a configurable overlap window (default 5 minutes)
so in-flight callers do not break. A new `rotate-secret` CLI under
`src/lib/vault_rotate.py` performs the swap, prints `old_version` and
`new_version`, and emits a structured audit log line. No service code
changes — services keep calling `secret(name)` as today. This keeps
the rotation primitive horizontal (sits inside ft002) rather than
duplicated per service.

## features
- [[ft002|vault-secrets]]

## data_model
No new tables. Adds an in-memory overlap cache inside the ft002 client
keyed by `(name, version)` with a TTL equal to the overlap window. The
audit log line is structured-stdout, not persisted by this story
(observability lives in cat004 and is out of scope here).

## api_surface
- `acme.lib.vault.secret(name)` — unchanged consumer signature; now
  returns the current version transparently and serves the previous
  version for `OVERLAP_SECONDS` after a rotation event.
- `acme.lib.vault_rotate.rotate(name, *, overlap_seconds=300)` —
  programmatic entry point.
- `python -m acme.lib.vault_rotate <name> [--overlap 300]` — CLI for
  operators. Exit 0 on success; non-zero with a clear stderr message
  on Vault errors. Stdout carries `{"name", "old_version",
  "new_version", "overlap_seconds"}` as JSON.
- New audit log line on stdout: `event=secret.rotated name=<n>
  old_version=<v1> new_version=<v2> actor=<user>`.

## components
- `src/lib/vault.py` — extend with dual-version cache + refresh hook.
- `src/lib/vault_rotate.py` — new module: rotate + CLI entry point.
- `src/services/auth/` `src/services/metrics/` `src/services/reports/`
  — no source changes; verified by synthetic check during rotation.
- `tests/lib/test_vault_rotate.py` — covers dual-version read,
  overlap window expiry, audit log shape, error paths.
