# Story [[platform-flow]] [[product]]

## role
[[pn002|platform-engineer]]

## want
rotate service credentials without downtime

## because
quarterly secret-rotation is currently a maintenance-window task, and
scheduling outages for every credential rotation blocks our SLA and
forces off-hours work on the platform team

## acceptance_criteria
- A rotation command can swap a named secret in Vault while the
  consuming services (auth, metrics, reports) remain serving traffic.
- The previous secret value stays valid for a 5-minute overlap window
  after rotation completes, so in-flight callers using the cached old
  value succeed.
- Synthetic check covering all three services observes zero 5xx
  responses during a rotation event.
- Operator can rotate a single secret by name (not all-or-nothing) and
  observes a clear success/failure result with the new version id.
- Rotation is auditable: each rotation emits a structured log line
  carrying secret name, old version, new version, actor, timestamp.
