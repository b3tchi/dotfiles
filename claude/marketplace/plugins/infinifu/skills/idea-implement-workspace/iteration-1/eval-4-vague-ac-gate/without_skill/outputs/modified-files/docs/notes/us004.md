# Story [[reports-flow]] [[product]]

## role
[[pn001|analyst]]

## want
search past reports by title substring and (optionally) restrict to a date range

## because
finding an old report by scrolling the 20-row dashboard is slow once
results age out of the recent list; analysts already know roughly what
they ran and when

## acceptance_criteria
- A search box on `/dashboard` accepts a free-text query and matches
  the report title case-insensitively (substring match)
- An optional date range narrows results to reports whose `run_at`
  falls within the window (inclusive both ends)
- Results show the same columns as the dashboard rows (title, status,
  run_at, CSV link) and respect the 90-day retention window
- Empty query + empty date range returns the unfiltered last-20 view
  (no change to current behavior)
- Search executes server-side against `report_runs` (not client-side
  over the 20 visible rows) so reports older than the dashboard window
  are reachable
- p95 latency ≤ 500 ms for a query over the full 90-day retention set
  on the current Postgres instance
- Only authenticated analysts (ft001) can search; unauthenticated
  requests return 401
