# Spec [[cat002]] [[board]]

## solves
[[us004|search reports by title and date]]

## implements
[[im002|server-side report search via /dashboard?q=&from=&to=]]

## problem
Analysts ([[pn001]]) can see the last 20 reports on `/dashboard`
([[im001]]) but cannot find older runs — reports persist for 90 days
([[adr0002]]) yet only the most recent slice is reachable from the
UI. Today re-finding a report means digging through email
notifications or asking engineering to run SQL. The story
([[us004]]) asks for a search box plus optional date range that
operates server-side against the full 90-day window with p95 latency
≤ 500 ms, authenticated through [[ft001]].

Goal: keep the dashboard's existing layout and auth, add `q` / `from`
/ `to` query params that translate into a single indexed SQL query,
return the same row shape as today. Out of scope: full-text ranking,
saved searches, search across columns other than title, exposing
search to non-analysts.
