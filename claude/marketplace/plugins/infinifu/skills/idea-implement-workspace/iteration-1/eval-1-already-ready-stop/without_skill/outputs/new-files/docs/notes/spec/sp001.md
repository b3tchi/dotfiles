# Spec [[cat002]] [[board]]

## solves
[[us002|filter reports by date range]]

## implements
[[im002|client-side date-range filter on the reports dashboard]]

## problem
[[pn001|analyst]] users currently land on `/dashboard` (shipped by
[[im001]]) and see the last 20 reports they ran — but at quarter-end
they need to narrow that list to a specific window (e.g. only runs
from the last week of Q1). Today they have to eyeball the timestamp
column and scroll, which is slow and error-prone when reviewing many
reports in a tight window.

Acceptance criteria from [[us002]]:

- Date picker on the dashboard.
- Filter applied client-side over the last 20 rows.
- Clearing the filter restores the full list.

Out of scope:

- Server-side pagination or filtering beyond the existing 20-row feed.
- Persisting the filter across sessions.
- Filter UI for other report-related pages.
