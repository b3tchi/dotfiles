# Implementation [[cat002]]

## solves
[[us002|filter reports by date range]]

## approach
Extend the existing `/dashboard` page (shipped by [[im001]]) with a
date-range picker. Filtering is client-side over the same 20 rows the
dashboard already loads via `GET /dashboard/data.json`, so no new
backend route, schema change, or query path is introduced. Honours
[[adr0002]] retention (90 days) implicitly by reusing the existing
data feed. Auth continues via [[ft001]] inherited from the dashboard
page.

## features
- [[ft001|basic-auth]]

## data_model
No schema change. Reads the same `report_runs` rows already exposed
through `/dashboard/data.json`.

## api_surface
No new endpoints. Existing `GET /dashboard/data.json` is consumed as-is.

## components
- `src/services/reports/templates/dashboard.html` (add picker markup)
- `src/services/reports/static/dashboard.js` (client-side filter logic)
