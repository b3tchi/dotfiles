# Feature [[cat001]] [[cat002]] [[product]]

## providing
Append-only ledger of destructive data events (today: hard-deletes
performed by retention jobs; future: secret revocation, account
disablement). Any service performing a hard-delete writes one row
per affected primary-key. Designed to outlive its first consumer
(us005) so secondary compliance asks (key rotation, account purges)
can plug in without re-inventing the schema.

## api_surface
```python
from acme.lib.audit_log import record_deletion

record_deletion(
    table="report_runs",
    primary_key="42",
    job_id="retention-2026Q1-batch-7",
    deleted_at=None,   # defaults to now(); UTC
)
```

A second helper exports a signed CSV slice:

```python
from acme.lib.audit_log import export_quarter

export_quarter(
    quarter="2026Q1",     # ISO quarter
    out_path="/tmp/2026Q1.csv",
    signing_key=secret("audit/signing-key"),
)
```

## data_model
Owns table `audit_deletions`:

| column         | type         | notes                                  |
|----------------|--------------|----------------------------------------|
| id             | bigserial PK | row id                                 |
| table_name     | text         | source table                           |
| primary_key    | text         | string-encoded PK of the deleted row   |
| job_id         | text         | triggering job / actor                 |
| deleted_at     | timestamptz  | UTC, defaults to `now()`               |
| recorded_at    | timestamptz  | UTC, set on insert                     |

Append-only by convention; no UPDATE / DELETE grants on the role
that writes it. Retained indefinitely (it *is* the retention
evidence — deleting it defeats the point).
