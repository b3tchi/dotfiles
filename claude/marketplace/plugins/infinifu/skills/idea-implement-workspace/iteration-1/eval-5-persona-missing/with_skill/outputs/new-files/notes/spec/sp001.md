# Spec [[cat001]] [[cat002]] [[board]]

## solves
[[us005|quarterly audit-log export]]

## problem

The [[pn003|compliance-officer]] persona needs a quarterly export of
all hard-deleted rows so legal can evidence proof-of-deletion to
auditors. Today no such export exists; deletions happen via the
nightly retention job ([[adr0002]] — Postgres `report_runs`, 90-day
hard delete) but leave no durable, queryable audit trail outside the
deletion itself.

Story [[us005]] requires a CSV of all rows hard-deleted in the
trailing quarter, with deletion timestamp, table, primary-key, and
triggering job id, signed with a vault-managed key for tamper-evidence.

**Categories touched.** This is both a security concern (audit trail,
tamper-evidence — [[cat001]]) and a data concern (retention,
deletion-record schema — [[cat002]]).

**Binding decisions to inherit.**

- [[adr0001]] — services authenticate via [[ft001]] basic-auth, so any
  export endpoint or CLI the compliance officer touches inherits that
  contract.
- [[adr0002]] — reports persist in Postgres and are hard-deleted after
  90 days via a nightly job. The deletion event itself is the data
  source for the audit log; capturing it durably (without inflating
  the 90-day window) is the core data-modeling question.

**Candidate feature consumers (binding deferred to spec-writing).**

- [[ft002]] — vault-secrets is the natural source of the tamper-
  evidence signing key. Spec-writing decides the exact key shape /
  rotation cadence.
- [[ft001]] — basic-auth gates whoever triggers the export
  (compliance-officer must authenticate even for read-only artifact
  generation).

No existing `[[ft###]]` provides an audit-log capability, so
spec-writing will likely propose either a new feature or a story-local
implementation; that choice is out of scope for the idea stage.

**Open shape questions (for spec-writing).**

- Where does the deletion record live? Sidecar table written by the
  nightly job, append-only event log, or reconstructed from WAL? The
  90-day retention from [[adr0002]] cannot apply to the audit trail
  itself or quarterly evidence vanishes.
- Synchronous CSV vs scheduled job depositing into a known location?
  The compliance-officer persona's `open_questions` flag this
  ([[pn003]] — UI vs CLI vs scheduled export).
- Signing scope: per-row HMAC, manifest digest, detached signature on
  the CSV file, or all three?
