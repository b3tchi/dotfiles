# Implementation [[cat001]] [[cat003]]

## solves
[[us003|rotate service credentials without downtime]]

## approach
Extend [[ft002]] vault-secrets with a dual-secret overlap window. Vault
holds `current` and `previous` versions of each secret simultaneously
during a configurable overlap (default 5 minutes). The `secret(name)`
read path transparently accepts either version while the window is
open; after the window, `previous` is destroyed. A new admin entry
point `rotate(name, new_value, overlap=300)` writes the rotation and
schedules cleanup. A `bin/rotate-secret` CLI wraps it for the
platform-engineer, gated by a synthetic check (curl loop against each
of the three services) that aborts rotation on any observed 5xx.

Chosen over vault-rotate-policy (too Vault-coupled, no overlap-window
control, nowhere natural for the synthetic-check hook) and over
blue-green-cred (heavy infra change, conflicts with [[adr0003]]'s
minimal-infra preference). Inherits the auth contract from [[adr0001]]
and leaves [[adr0002]] data retention untouched (rotation is
credential-only).

## features
- [[ft002|vault-secrets]]

## data_model
No schema change. The dual-secret state lives in Vault, not Postgres.
[[adr0002]] `report_runs` table is untouched.

## api_surface
Widening of [[ft002]] (not a new service):

- `secret(name)` — unchanged signature, now reads `current` and
  transparently accepts `previous` during an open overlap window.
- `rotate(name, new_value, overlap=300)` — new admin entry point.
  Writes `new_value` as `current`, demotes the old to `previous`,
  schedules destruction of `previous` after `overlap` seconds.
- `bin/rotate-secret <name> <new_value> [--overlap 300]` — CLI for
  the platform-engineer. Runs the synthetic check before+during+after
  the swap; aborts and rolls back on observed 5xx.

## components
- `src/lib/vault.py` — widen `secret()`, add `rotate()`
- `bin/rotate-secret` — new CLI entry point (story-specific glue)
- synthetic-check helper (path pinned at spec-refinement)
