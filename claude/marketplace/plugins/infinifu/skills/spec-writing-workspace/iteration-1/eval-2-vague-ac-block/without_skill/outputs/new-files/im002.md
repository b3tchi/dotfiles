# Implementation [[cat001]] [[cat003]]

## solves
[[us003|rotate service credentials without downtime]]

## approach
Add a versioned (`current` + `previous`) read path inside
[[ft002|vault-secrets]] and a new `acme-rotate` CLI that drives a
5-minute overlap window: mint new value → write `current` (demoting
old to `previous`) → poll a per-service `/healthz/auth` synthetic →
expire `previous` after the window. Reuses the existing
`secret(name)` API surface so no consumer code changes. Inherits
[[adr0001]] (basic-auth scope) and [[adr0002]] (Postgres retention)
unchanged.

## features
- [[ft002|vault-secrets]]

## data_model
No new tables. Vault stores `current` and `previous` versions per
secret path; the dual-version state is internal to Vault, not surfaced
to callers.

## api_surface
- `acme-rotate <secret-name> [--overlap=5m] [--dry-run]` — new CLI.
- `GET /healthz/auth` — new synthetic-check endpoint per service.
- `secret(name)` — unchanged (returns the scalar `current` value).

## components
- `src/lib/vault_rotate.py` — orchestrator + CLI entry point.
- `src/lib/vault.py` — extended with version-aware helpers.
- `src/services/*/healthz.py` — per-service synthetic-check endpoint.
