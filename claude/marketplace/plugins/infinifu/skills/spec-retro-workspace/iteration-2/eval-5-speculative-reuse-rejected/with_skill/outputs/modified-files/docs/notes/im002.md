# Implementation [[cat001]] [[cat003]]

## solves
[[us003|rotate service credentials without downtime]]

## approach
Adopt the vault-rotate-policy pattern via [[ft002]]: writers stage the
new credential as an additional versioned alias inside the in-process
vault state; readers continue to call `vault.secret(name)` and receive
the most recently staged version. Per-name serialization is provided by
a `threading.Lock` keyed by secret name so concurrent rotations of the
same secret cannot lose a write or interleave. Empty / `None` values
are rejected at the API boundary with `ValueError` before any state
mutation, and backend failures are wrapped in `VaultError` so callers
get a single typed failure surface.

Shipped scope is the `rotate_secret` write helper plus the input-validation
and per-name serialization that the success criteria of Task 1 required;
audit closed all three tasks as approved against the success criteria
that were actually exercised by the shipped tests.

## features
- [[ft002|vault-secrets]]

## data_model
No schema change. Vault state is an in-process map of `name → [versions]`
list; the latest element of the list is the value `secret(name)` returns.
The list is mutated only under the per-name lock.

## api_surface
- `acme.lib.vault.secret(name) -> str` — read path (unchanged, ft002 surface).
- `acme.lib.vault.rotate_secret(name: str, new_value: str) -> None` — write
  path. Internal to this Implementation. Raises `ValueError` on empty
  name or value, `VaultError` on backend failure.

`rotate_secret` is deliberately kept as a vertical helper inside this
Implementation. It is not promoted to [[ft002]]'s `## api_surface` and
no new `ft###` is minted for it — see retro notes for the
vertical-over-horizontal rationale.

## components
- `src/lib/vault.py` — `secret`, `rotate_secret`, `VaultError`, the
  per-name lock registry, and the in-process alias map.
- `tests/lib/test_vault.py` — five tests covering alias staging,
  post-flip reads, concurrent serialization, empty-value rejection, and
  empty-name rejection.
