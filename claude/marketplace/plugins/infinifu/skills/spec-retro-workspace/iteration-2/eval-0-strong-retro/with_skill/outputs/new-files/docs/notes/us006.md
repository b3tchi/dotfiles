# Story [[platform-flow]] [[product]]

## role
[[pn002|platform-engineer]]

## want
verify credential rotation behaves correctly across regions

## because
the synthetic check that gates rotation today is single-region, so
rotation correctness across regions is unverified — a regional failover
during a rotation window could surface 5xx that the existing
[[sp001]] check would miss

## acceptance_criteria
- Synthetic check during a rotation window runs against more than one
  region (at minimum: primary + one secondary)
- Cross-region read-after-rotate returns the new value within the
  overlap window (no stale reads from a lagging region)
- Forced regional failover mid-rotation does not produce 5xx beyond the
  single-transient threshold already established in [[us003]]
- Test plan exercises the failover path end-to-end, not just the
  happy-path multi-region read
