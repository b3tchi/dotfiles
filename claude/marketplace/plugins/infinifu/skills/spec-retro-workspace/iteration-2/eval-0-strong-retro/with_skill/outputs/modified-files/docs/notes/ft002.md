# Feature [[cat001]] [[product]]

## providing
Vault-backed secret retrieval *and* live rotation. Every service calls
`secret(name)` to read credentials at runtime; rotation writers call
`rotate_secret(name, new_value)` to stage a new versioned alias under
per-name serialization. Errors surface as `VaultError`.

## api_surface
```python
from acme.lib.vault import secret, rotate_secret, VaultError

# read path
db_url = secret("reports/db_url")

# rotation path (per-name serialized; raises ValueError on empty input,
# VaultError if the backend rejects)
rotate_secret("reports/db_url", new_value)
```

## data_model
None local. Vault is the source of truth. In-memory the client keeps a
per-name version list so `secret(name)` returns the most recently
staged value; older versions remain reachable until expiry lands.

## sample
`src/lib/vault.py` ships the canonical client (read + rotate).
