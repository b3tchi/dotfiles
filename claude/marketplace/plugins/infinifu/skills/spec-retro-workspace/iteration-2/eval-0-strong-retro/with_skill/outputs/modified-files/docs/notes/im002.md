# Implementation [[cat001]] [[cat003]]

## solves
[[us003|rotate service credentials without downtime]]

## approach
Extend [[ft002|vault-secrets]] with an in-process rotation primitive.
`rotate_secret(name, new_value)` stages a new versioned alias under a
per-name `threading.Lock` (lazily created and guarded by a single
`_LOCKS_GUARD`), so concurrent writers for the same name serialize but
distinct names rotate in parallel. `secret(name)` reads the most
recently staged version, so readers transparently see the new value as
soon as it lands. Empty `name` or empty/`None` `new_value` is rejected
at the API boundary with `ValueError`; backend failures surface as
`VaultError`. The planned 5-minute overlap-then-expire orchestration
(separate `vault_rotate` scheduler) did *not* ship in this spec and is
captured for follow-up — today the staging primitive is the surface,
and expiry is left to a subsequent iteration.

## features
- [[ft002|vault-secrets]]

## data_model
No schema change. Vault remains the source of truth; in-process the
client keeps a per-name list of staged values (`_ALIASES: dict[str,
list[str]]`) so `secret(name)` returns the last entry.

## api_surface
Public surface lives on `acme.lib.vault`:

- `secret(name) -> str` — read path; returns the most recently staged
  version, or a sentinel `<vault:{name}>` when no rotation has occurred.
- `rotate_secret(name, new_value) -> None` — stages a new versioned
  alias under a per-name lock. Raises `ValueError` on empty name /
  empty / `None` value; raises `VaultError` if the backend rejects.
- `VaultError` — re-exported exception type consumers catch.

No `vault_rotate.rotate(...)` orchestration entrypoint shipped; the
scheduler module remains future work.

## components
- `src/lib/vault.py` — `secret`, `rotate_secret`, `VaultError`,
  `_lock_for`, `_read_alias`, module-level `_LOCKS` / `_LOCKS_GUARD` /
  `_ALIASES` state.
- `tests/lib/test_vault.py` — five-test contract: stages new alias,
  post-flip read, concurrent serialization (5 threads), empty-value
  rejection, empty-name rejection.
