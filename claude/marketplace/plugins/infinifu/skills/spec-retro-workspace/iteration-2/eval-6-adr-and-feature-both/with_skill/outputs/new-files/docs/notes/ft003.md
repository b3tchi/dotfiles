# Feature [[cat003]] [[product]]

## providing
Thin, reusable wrapper around HashiCorp Vault's Transit secrets engine.
Exposes deterministic encrypt / decrypt / rotate_key primitives so any
service that needs encrypted versioned rotation can call into one place
instead of rebuilding the wrapper. Depends on [[ft002|vault-secrets]]
for Vault connectivity; the Transit surface is additive, not a
replacement.

Binds [[adr0004|Vault Transit secrets engine over plain KV for credential rotation]].

## api_surface
```python
from acme.lib.vault_transit import encrypt, decrypt, rotate_key

ciphertext = encrypt("plaintext", key_name="alerts-signing")
plaintext  = decrypt(ciphertext, key_name="alerts-signing")
new_version = rotate_key("alerts-signing")  # returns new version number
```

## data_model
None local. Transit keys live in Vault under the named key namespace.
Key versions are owned by Transit; consumers never write key material
directly.

## sample
`src/lib/vault_transit.py` ships the canonical wrapper. The
[[im002|vault-policy credential rotation]] flow consumes it to back
versioned aliases for the 5-minute overlap window.
