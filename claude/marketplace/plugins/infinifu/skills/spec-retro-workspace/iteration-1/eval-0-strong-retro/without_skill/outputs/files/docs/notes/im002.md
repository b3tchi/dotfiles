# Implementation [[cat001]] [[cat003]]

## solves
[[us003|rotate service credentials without downtime]]

## approach
Adopt the vault-rotate-policy pattern via [[ft002]]: writers stage the
new credential under a versioned alias via `vault.rotate_secret`;
readers via `vault.secret` always see the most recent staged version.
Orchestration (5-minute overlap timer, expiry, synthetic-check hook)
was scoped in [[sp001]] but not implemented in this card — see retro
follow-ups for the deferred work.

## features
- [[ft002|vault-secrets]]

## data_model
No schema change. Vault holds versioned aliases.

## api_surface
`acme.lib.vault.secret(name)` continues to be the only read path; writes
use a new internal `rotate_secret(name, new_value)` helper in the same
lib (per-name lock, rejects empty input, raises `VaultError` on backend
failure).
