# Story [[platform-flow]] [[product]]

## role
[[pn002|platform-engineer]]

## want
verify a credential rotation propagates correctly across all regions
before the 5-minute overlap window expires

## because
the synthetic check shipped with [[sp001]] is single-region; a
multi-region service could see stale credentials in one region while
another already flipped, defeating the zero-5xx guarantee

## acceptance_criteria
- Synthetic check runs in every region the service is deployed to
- Cross-region read of a rotated secret returns the new value within
  the overlap window
- Integration test exercises a two-region simulated rotation and
  asserts no 5xx in either region
