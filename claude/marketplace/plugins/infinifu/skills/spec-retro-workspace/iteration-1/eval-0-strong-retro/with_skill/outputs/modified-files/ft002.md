# Feature [[cat001]] [[product]]

## providing
Vault-backed secret retrieval and live rotation. Every service calls
`secret(name)` to read credentials at runtime; operators call
`rotate_secret(name, new_value)` to stage a new versioned alias without
downtime — readers continue to call `secret(name)` and transparently
get the latest staged version.

## api_surface
```python
from acme.lib.vault import secret, rotate_secret, VaultError

# Read
db_url = secret("reports/db_url")

# Rotate (per-name serialized; raises ValueError on empty input,
# VaultError on backend failure)
rotate_secret("reports/db_url", "<new-value>")
```

## data_model
None local in the contract. The current implementation keeps an
in-memory `_ALIASES` map of staged versions per name plus a per-name
lock map; backend vault remains the source of truth for storage and
version retention.

## sample
`src/lib/vault.py` ships the canonical client (read + rotate live in
the same module — see [[im002]] and [[adr0004]]).
