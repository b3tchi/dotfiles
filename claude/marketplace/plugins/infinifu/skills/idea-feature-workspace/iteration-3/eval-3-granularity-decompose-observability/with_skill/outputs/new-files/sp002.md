# Spec [[cat004]] [[board]]

## problem

We need a shared **structured logging** capability — a horizontal
building block that every service uses to emit JSON log records with a
consistent schema (timestamp, level, service, trace_id, message,
attributes) and ship them to a central sink. Today every service writes
free-form text to stdout, which is mentioned explicitly in
`src/services/metrics/__init__.py` ("Alert path currently writes to
stdout") and is the same pattern in `auth` and `reports`.

**No existing `ft###` covers this.** Surveyed features: [[ft001]]
(basic-auth) and [[ft002]] (vault-secrets) — both [[cat001]] security,
unrelated. We'd mint a new `ft###` at spec-writing time.

**Granularity.** This sp### scopes **one** ft### — the structured-logger
client. Companion observability features ride in [[sp001]] (metrics),
[[sp003]] (tracing), [[sp004]] (alerting), [[sp005]] (dashboards). Each
is its own atomic ft### so the JSON schema and the ship target (Loki?
Elasticsearch? local file?) can be locked independently of metrics or
tracing decisions.

**Consumers (via existing `im###`).** Surveyed [[im001]] (reports
dashboard) — currently uses no shared logger, candidate consumer. The
three running services are all migration targets — every `print(...)`
or `stdout` write across `src/services/auth/`, `src/services/metrics/`,
`src/services/reports/` becomes a `log.info(...)` call once the feature
exists.

**Category.** [[cat004]] (observability) is the primary; the structured
schema also touches [[cat002]] (data) if retention and indexing become a
concern.

**Binding ADRs.** No accepted ADR under [[cat004]] yet. [[adr0002]]
(reports retention = 90 days) suggests a similar bounded-retention
policy on log storage — the new feature should declare a retention
constraint up front to inherit the same operating mindset.

**Migration intent.** At spec-writing time, mint `ft###` for the
structured logger; replace ad-hoc stdout writes in all three services
with the shared logger; back-fill log calls where today's code emits
nothing.
