# Story [[platform-flow]] [[product]]

## role
[[pn002|platform-engineer]]

## want
run a quarterly retention purge tool that hard-deletes data past the
retention window and emits a signed deletion-proof report

## because
legal asks once a quarter for evidence that retention rules
(see [[adr0002]] — 90-day report retention) were enforced, and today
the platform-engineer either trusts the nightly cron or stitches a
proof together by hand

## acceptance_criteria
- A CLI / one-shot tool exists that the platform-engineer can run
  on-demand (not a service, not on a schedule beyond the existing
  nightly cron)
- The tool identifies rows past the [[adr0002]] retention window in
  scope (initially `report_runs`)
- The tool hard-deletes those rows and writes a deletion-proof
  artifact (counts, ids, timestamps, operator, table) somewhere
  legal can be handed
- The deletion-proof artifact is reproducible: re-running the tool
  the same day produces an empty proof, not a duplicate purge
- Only [[pn002|platform-engineer]] can invoke it (auth-gated via
  [[ft001]] or equivalent operator-only path)
