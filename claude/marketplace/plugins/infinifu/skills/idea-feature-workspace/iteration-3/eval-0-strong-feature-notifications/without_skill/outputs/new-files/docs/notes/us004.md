# Story [[reports-flow]] [[product]]

## role
[[pn001|analyst]]

## want
get notified when a long-running report finishes

## because
I do not want to keep re-checking the dashboard during multi-minute
or overnight runs; today this is handled by ad-hoc smtplib in the
reports service

## acceptance_criteria
- Notification fires once per finished report, at most once per run
- Notification carries the report id and a link back to the dashboard
- Channel preference (email today, slack soon) is configurable per
  analyst rather than hard-coded in the reports service
- Failures to deliver the notification do not block the report job
  from completing
